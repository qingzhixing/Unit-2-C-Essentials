## Lesson 40 — 二分查找【重点】

### 课程任务

实现迭代二分查找 `binary_search(arr, n, target)`，在**有序**数组中查找目标值，找到返回索引，未找到返回 -1。

```c
int binary_search(int arr[], int n, int target);
```

验证用例：
```
输入 "1 3 5 7 9 11" + "\n5\n"  → 2
输入 "1 3 5 7 9 11" + "\n6\n"  → -1
输入 "1"            + "\n1\n"  → 0
```

二分查找是算法面试中**出现频率最高的题目之一**。Jon Bentley 在《编程珠玑》中提到，他让 100 多位专业程序员写二分查找，90% 写出的代码有 bug——多数栽在三个经典陷阱上。

### 前置知识：二分查找的核心思想

二分查找的核心假设：**数组有序**。为什么有序是必须的？因为每次比较中间元素后，需要推断 target 在左半还是右半。如果数组无序，`arr[mid] < target` 不能推出 target 一定在右侧——它可能散布在任意位置。

**为什么叫"二分"？** — 每次比较后，搜索空间减半：

```
数组 [1, 3, 5, 7, 9, 11]，查找 5:

第1轮: lo=0, hi=5, mid=2 → arr[2]=5 == target → 找到!
┌───┬───┬───┬───┬───┬───┐
│ 1 │ 3 │ 5 │ 7 │ 9 │11 │    只比较 1 次! (线性查找需要 3 次)
└───┴───┴───┴───┴───┴───┘
  ↑        ↑          ↑
 lo=0    mid=2       hi=5

数组 [1, 3, 5, 7, 9, 11]，查找 9:

第1轮: lo=0, hi=5, mid=2 → arr[2]=5 < 9 → lo=3（丢左半）
┌───┬───┬───┬───┬───┬───┐
│   │   │   │ 7 │ 9 │11 │    搜索空间: [3..5]
└───┴───┴───┴───┴───┴───┘
              ↑    ↑    ↑
             lo=3 mid=4 hi=5

第2轮: arr[4]=9 == target → 找到!
```

**线性 vs 二分直观对比**：
```
n=1000:   线性平均 500 次比较, 二分最多 10 次 (2^10=1024)
n=10^6:   线性平均 50 万次,     二分最多 20 次 (2^20≈10^6)
n=10^9:   线性平均 5 亿次,      二分最多 30 次 (2^30≈10^9)
```

### 算法详解：三大经典陷阱

二分查找不到 10 行代码，但三个细节能有效区分"背代码"和"真理解"。

#### 陷阱 1：mid 计算溢出（最隐蔽的 bug）

错误写法只在大数组时触发，平时测试发现不了：

```c
/* ❌ 错误: lo+hi 可能超过 INT_MAX */
int mid = (lo + hi) / 2;

/* ✅ 正确: hi-lo 永远不会溢出 */
int mid = lo + (hi - lo) / 2;
```

**溢出演示**（32 位 int，INT_MAX = 2147483647）:

```
lo = 1500000000, hi = 2000000000

(lo + hi) / 2 = 3500000000 / 2      → lo+hi > INT_MAX → 溢出!
               → 3500000000 被解释为负数 -794967296
               → mid = -397483648    → 错误索引!

lo + (hi-lo)/2 = 1500000000 + 250000000 = 1750000000 → 正确! ✓
```

Java 的 `Arrays.binarySearch` 在 2006 年前一直有这个 bug（Bug ID 5045582），潜伏了 9 年。Google 的 TimSort 实现也曾在 2015 年因类似溢出导致崩溃。

#### 陷阱 2：循环条件 `lo <= hi` vs `lo < hi`

| 写法 | 行为 | 后果 |
|------|------|------|
| `while (lo <= hi)` ✅ | lo==hi 时仍进入循环 | 正确处理单元素搜索空间 |
| `while (lo < hi)` ❌ | lo==hi 时直接退出 | 漏掉单元素情况! |

```
数组 [5]，查找 5:

while (lo <= hi): lo=0, hi=0 → 进入循环 → mid=0 → arr[0]==5 → 返回 0 ✓

while (lo < hi):  lo=0, hi=0 → 不进入循环 → 返回 -1 ✗ (明明有 5!)
```

#### 陷阱 3：指针更新 `lo = mid + 1` 而非 `lo = mid`

```
/* ❌ 错误: 死循环! */
if (arr[mid] < target)
    lo = mid;      // 当 lo=0, hi=1, mid=0 时, lo 永远是 0!
    
/* ✅ 正确 */
if (arr[mid] < target)
    lo = mid + 1;  // lo 必然前进
```

### 逐步跟踪：查找 5 in [1, 3, 5, 7, 9, 11]

```
arr = [1, 3, 5, 7, 9, 11], n=6, target=5

轮次 | lo | hi | mid=lo+(hi-lo)/2 | arr[mid] | 判断 | 操作
1    | 0  | 5  | 0+(5-0)/2=2      | 5        | ==5  | return 2 ✓
```

### 查找不存在值：查找 6 in [1, 3, 5, 7, 9, 11]

```
轮次 | lo | hi | mid | arr[mid] | 判断      | 操作
1    | 0  | 5  | 2   | 5        | 5<6       | lo=3
2    | 3  | 5  | 4   | 9        | 9>6       | hi=3
3    | 3  | 3  | 3   | 7        | 7>6       | hi=2
4    | 退出: lo(3) > hi(2) → return -1 ✓
```

### 边界条件表

| 场景 | 预期 | 原因 |
|------|------|------|
| target 在开头 | 找到 | lo=0 起始 |
| target 在末尾 | 找到 | `<=` 等号保证不遗漏 |
| target 不存在 | -1 | lo > hi 退出 |
| 空数组 (n=0) | -1 | hi=-1, 不进入循环 |
| 单元素+匹配 | 0 | lo=hi 进入循环 |
| 单元素+不匹配 | -1 | 更新后 lo>hi |

### 复杂度分析

| 指标 | 值 | 说明 |
|------|-----|------|
| 时间 | O(log n) | 每次折半，n/(2^k)=1 → k=log₂n |
| 空间 | O(1) | 只用了 lo/hi/mid 三个变量 |
| 比较次数 | ≤⌊log₂n⌋+1 | 最坏情况 |

### lower_bound 变体（扩展）

标准二分查找找"等于 target"。`lower_bound` 找"第一个 ≥ target"：

```
数组 [1, 3, 3, 5, 7], lower_bound(3) = 1 (第一个 3)
                     binary_search(3) = 1 或 2 (任意一个 3)

关键差异: hi=n (不是 n-1), while(lo<hi) (不是 <=), hi=mid (不是 mid-1)
```

### 常见错误

| 错误 | 后果 | 正确 |
|------|------|------|
| `mid = (lo+hi)/2` | 大数组溢出 | `mid = lo+(hi-lo)/2` |
| `while(lo < hi)` | 漏单元素 | `while(lo <= hi)` |
| `lo = mid` 而非 `mid+1` | 死循环 | `lo = mid+1` |
| 忘记数组有序前提 | 结果不确定 | 仅用于有序数组 |
| 返回 `arr[mid]` | 返回元素值而非索引 | `return mid;` |

### 重要知识点

- 二分查找 O(log n)——每轮搜索空间减半，是对数复杂度的典范
- 三大陷阱：mid 溢出、循环条件漏等号、指针更新用 mid 而非 mid±1
- `mid = lo + (hi - lo) / 2` 是防溢出的标准写法
- 前提是数组有序——无序数组只能用线性查找

### 课堂讨论

1. **lower_bound 和标准二分有什么区别？** — lower_bound 找"第一个 ≥ target"的位置，hi=n（允许返回 n），循环用 `lo<hi`（不用 `<=`），hi=mid（不用 mid-1）。这些差异都源于目标不同：找左边界 vs 找精确匹配。

2. **二分查找能用递归实现吗？** — 能。递归版代码更短，但每次调用消耗栈空间。迭代版 O(1) 空间，在面试中更受青睐。两者时间复杂度完全相同。

3. **旋转有序数组怎么二分？** — 如 `[4,5,6,7,0,1,2]`，先判断 mid 落在左上升段还是右上升段，再决定搜索哪一侧。本质是标准二分的变体——每次仍能排除一半。

4. **二分查找的时间复杂度为什么是 O(log n)？** — 每轮搜索空间除以 2：n→n/2→n/4→...→1，共 log₂n 轮。每轮 O(1) 比较，总 O(log n)。

5. **如果数组中有重复元素，标准二分返回的是第几个？** — 不确定。标准二分在 arr[mid]==target 时立即返回 mid，不保证是第一个还是最后一个。需要稳定语义时用 lower_bound/upper_bound。
