## Lesson 35 — 环形队列基础

### 课程任务

实现环形队列的两个核心操作：

1. **`enqueue(val)`** — 入队：将元素放入队尾，rear 指针前移（通过取模 `% MAX` 实现环绕）
2. **`dequeue()`** — 出队：从队头取出元素，front 指针前移（通过取模实现环绕）

`is_empty` 和全局变量 `queue[MAX]`、`front`、`rear` 已由模板提供。你的任务是实现 `enqueue` 和 `dequeue` 两个函数。

```c
#define MAX 64
int queue[MAX];
int front = 0, rear = 0;   // 初始状态: front==rear 表示队空

int is_empty(void) { return front == rear; }
```

### 前置知识：什么是队列？为什么需要"环形"？

#### 队列的 FIFO 语义

队列（Queue）是一种**先进先出**（FIFO, First In First Out）的数据结构。像排队买票——先到的人先服务，后来的人排在队尾。

```
入队顺序: A → B → C
出队顺序: A → B → C  (先进先出!)
```

队列的两个基本操作：
- **enqueue（入队）**: 将元素添加到队尾
- **dequeue（出队）**: 从队头取出元素

#### 线性队列的"假溢出"问题

如果不取模，仅让 front/rear 单调递增：

```
MAX = 5 的线性队列:

初始:    [_,_,_,_,_]  front=0, rear=0

入队 A:  [A,_,_,_,_]  front=0, rear=1
入队 B:  [A,B,_,_,_]  front=0, rear=2
入队 C:  [A,B,C,_,_]  front=0, rear=3
出队:    [_,B,C,_,_]  front=1, rear=3   ← A 出队
出队:    [_,_,C,_,_]  front=2, rear=3   ← B 出队
入队 D:  [_,_,C,D,_]  front=2, rear=4
入队 E:  [_,_,C,D,E]  front=2, rear=5

rear == MAX → "满了"！

但实际上 [0] 和 [1] 是空的！front 前面的空间被浪费了。
这就是"假溢出"——数据并没有满，但 rear 已经走到数组尽头，
无法继续入队。
```

#### 环形队列的核心思想

当指针走到数组末尾时，通过取模运算 `% MAX` 绕回数组开头，重用已被出队的空间。

```
把数组想象成一个环:

        ┌───┐
    ┌─→ │ 0 │ ──┐
    │   └───┘   │
    │   ┌───┐   │
    │   │ 1 │   │
    │   └───┘   │    在这个环上，front 和 rear 顺时针移动
    │   ┌───┐   │    (ptr + 1) % MAX 就是"向前一步"
    │   │ 2 │   │
    │   └───┘   │
    │   ┌───┐   │
    │   │...│   │
    │   └───┘   │
    │   ┌───┐   │
    └── │63 │ ←─┘
        └───┘

当 rear=63 时, (63+1) % 64 = 0 → 绕回位置 0!
```

### 核心操作详解

```
入队: queue[rear] = val;   // ① 先把值放到 rear 指向的位置
      rear = (rear + 1) % MAX;  // ② rear 前移（取模环绕）

出队: val = queue[front];       // ① 先从 front 指向的位置取值
      front = (front + 1) % MAX; // ② front 前移（取模环绕）
```

**为什么入队"先放值再移指针"？**

```
初始: front=0, rear=0

方案A（先移再放）:  rear = (0+1)%MAX = 1
                  queue[1] = val   ← 位置 0 被跳过了！永远不会用

方案B（先放再移）:  queue[0] = val  ← 位置 0 被正确使用 ✓
                  rear = (0+1)%MAX = 1
```

### 状态判定

| 状态 | 条件 | 说明 |
|------|------|------|
| 队空 | `front == rear` | 初始状态，或全部出队后 |
| 非空 | `front != rear` | 至少有一个元素 |
| 队满 | `(rear + 1) % MAX == front` | rear 即将追上 front |

**为什么队满要牺牲一个位置？**

```
问题: front==rear 既表示队空又表示队满，无法区分。

解决: 约定 rear 永远不"追上" front。
     队空: front == rear
     队满: (rear+1) % MAX == front  ← rear 在 front 前一步

代价: 实际容量 = MAX - 1
```

### 常见错误与陷阱

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 忘取模 `rear++` 代替 `(rear+1)%MAX` | 数组越界 | 必须用取模 |
| 入队先移 rear 再赋值 | 跳过位置 0 | `queue[rear]=val; rear=(rear+1)%MAX;` |
| dequeue 忘取模 | front 无限增长 | `front=(front+1)%MAX` |
| 混淆 front/rear 语义 | 逻辑完全错乱 | front=队头(出队处), rear=队尾下一个空位 |

### 重要知识点

- `(ptr+1) % MAX` 是环形结构的核心公式
- front 指向队头（下一个要出队的元素），rear 指向下一个空位
- 初始 `front=rear=0` 同时满足队空条件
- 牺牲一个位置换取队空/队满的简洁区分
- 环形队列工程应用极广：Linux kfifo、音频环形 buffer、网络数据包缓冲、生产者-消费者模式

### 课堂讨论

1. **如果想存满 MAX 个元素，怎么改？** — 增加 `count` 变量。入队 count++，出队 count--。队空: count==0，队满: count==MAX。不再需要牺牲一个位置。

2. **取模运算 `%` 性能如何？** — 取模是除法指令，比加减慢。高性能场景（如每秒百万次操作）可将 MAX 设为 2^n（如 64、128），用位运算 `(ptr+1) & (MAX-1)` 替代 `%`。

3. **环形队列线程安全吗？** — 默认不是。需要互斥锁保护或使用原子操作。Linux 内核 kfifo 用内存屏障实现了单生产者/单消费者的无锁环形缓冲——这是高性能网络栈的核心组件。

4. **和栈（LIFO）各适合什么场景？** — 栈适合回溯（DFS）、撤销操作、括号匹配；队列适合公平调度（BFS）、缓冲（削峰填谷）、消息队列。
