## Lesson 28 — 全排列问题：回溯算法（DFS）入门

### 课程任务

给定一个字符串，输出其中所有字符的全排列，每行一个。

实现 `permute(str, l, r)` 函数，其中 `swap` 辅助函数已提供。这是**回溯算法（Backtracking）**的经典入门题，也是后续八皇后（Lesson 29）和 DFS 迷宫（Lesson 43）的基础。

### 前置知识：什么是排列？什么是回溯？

**排列（Permutation）**：将一组元素按不同顺序排列。3 个字符 `A,B,C` 有 3! = 6 种排列：

```
ABC  ACB  BAC  BCA  CAB  CBA
```

**回溯（Backtracking）**：一种系统地搜索问题解的算法。核心思想是：逐步构建解，每走一步都检查是否可能通向一个完整解。如果当前路径不可能成功，就**退回（回溯）**上一步，尝试其他选择。

回溯与暴力枚举的本质区别在于**状态恢复**——在尝试一个分支后，必须恢复现场，才能正确尝试下一个分支。

### 算法详解：Swap-Based 回溯

#### 核心思路

递归地确定每个位置放哪个字符。位置 `l` 上的字符可以从 `l`, `l+1`, ..., `r` 中任意选择：

```
permute(str, l, r):
  若 l == r: 所有位置已确定，输出 str（找到一个解）
  否则:
    对 i 从 l 到 r:
      swap(str[l], str[i])     // 把候选字符换到位置 l
      permute(str, l+1, r)     // 递归确定后面的位置
      swap(str[l], str[i])     // 换回来！（回溯的关键）
```

**为什么需要 swap 两次？** 第一次 swap 把候选字符放到固定位置；递归调用探索以该候选开头的所有排列；第二次 swap **恢复原状**，让下一轮循环能在一个干净的字符串上尝试另一个候选。这就是回溯的"撤销"操作。

#### 排列树：递归的完整视角

以 `ABC` 为例，整个搜索过程形成一棵**排列树**：

```
                        位置 0                   位置 1           位置 2  (叶子)
                        ┌───┐
                        │ABC│  l=0, 所有位置待定
                        └─┬─┘
            ┌─────────────┼─────────────┐
        i=0:A↔A        i=1:A↔B        i=2:A↔C
        ┌───┐           ┌───┐           ┌───┐
        │ABC│           │BAC│           │CBA│  l=0 已定
        └─┬─┘           └─┬─┘           └─┬─┘
      ┌───┴───┐       ┌───┴───┐       ┌───┴───┐
   i=1:B↔B  i=2:B↔C i=1:A↔A  i=2:A↔C i=1:B↔B  i=2:B↔A
   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
   │ABC│   │ACB│   │BAC│   │BCA│   │CBA│   │CAB│  l=1 已定
   └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘   └─┬─┘
  i=2:C↔C  i=2:B↔B  i=2:C↔C  i=2:A↔A  i=2:A↔A  i=2:B↔B
   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
   │ABC│   │ACB│   │BAC│   │BCA│   │CBA│   │CAB│  l=2 == r, 输出!
   └───┘   └───┘   └───┘   └───┘   └───┘   └───┘
   ↑       ↑       ↑       ↑       ↑       ↑
   第1个解 第2个解 第3个解 第4个解 第5个解 第6个解
```

**关键观察**：
- 树的高度 = 字符串长度（每个位置一层）
- 叶子节点 = 完整排列，叶子数 = n!
- 每个内部节点的分支数递减（第 0 层 n 个分支，第 1 层 n-1 个，...）
- DFS 遍历顺序：沿最左分支走到底，然后回溯

#### 单步跟踪：从 ABC 到 ACB 的完整过程

```
字符串初始: "ABC", l=0, r=2

═══════════════════════════════════════════════════
第一轮: l=0, i=0 — 把 A 放到位置 0（A 本来就在位置 0）
═══════════════════════════════════════════════════

  swap(&str[0], &str[0])     →  字符串: "ABC"  (A↔A，无变化)
  permute("ABC", 1, 2)       →  递归进入 l=1
    ┌─ l=1, i=1: swap(&str[1],&str[1]) → "ABC" (B↔B)
    │  permute("ABC", 2, 2) → l==2==r → 输出 "ABC" ✓ (第1个解)
    │  swap(&str[1],&str[1]) → "ABC" (恢复)
    │
    ├─ l=1, i=2: swap(&str[1],&str[2]) → "ACB" (B↔C)
    │  permute("ACB", 2, 2) → l==2==r → 输出 "ACB" ✓ (第2个解)
    │  swap(&str[1],&str[2]) → "ABC" (恢复，回溯！)
    │
    └─ 循环结束，返回
  swap(&str[0], &str[0])     →  字符串: "ABC" (恢复)

═══════════════════════════════════════════════════
第二轮: l=0, i=1 — 把 B 换到位置 0
═══════════════════════════════════════════════════

  swap(&str[0], &str[1])     →  字符串: "BAC"  (A↔B)
  permute("BAC", 1, 2)       →  递归进入 l=1
    ┌─ l=1, i=1: swap(&str[1],&str[1]) → "BAC" (A↔A)
    │  permute("BAC", 2, 2) → 输出 "BAC" ✓ (第3个解)
    │  swap(&str[1],&str[1]) → "BAC" (恢复)
    │
    ├─ l=1, i=2: swap(&str[1],&str[2]) → "BCA" (A↔C)
    │  permute("BCA", 2, 2) → 输出 "BCA" ✓ (第4个解)
    │  swap(&str[1],&str[2]) → "BAC" (恢复)
    │
    └─ 循环结束，返回
  swap(&str[0], &str[1])     →  字符串: "ABC" (恢复，回溯！)

═══════════════════════════════════════════════════
第三轮: l=0, i=2 — 把 C 换到位置 0
═══════════════════════════════════════════════════

  (类似过程，输出 CBA 和 CAB)

  最终输出顺序: ABC → ACB → BAC → BCA → CBA → CAB
```

### 回溯三步曲

| 步骤 | 代码 | 含义 | 如果不做会怎样 |
|------|------|------|---------------|
| 1. 做选择 | `swap(&str[l], &str[i])` | 将候选字符换到当前固定位置 | 无法改变排列顺序 |
| 2. 递归 | `permute(str, l+1, r)` | 解决规模更小的子问题 | 无法深入探索 |
| 3. 撤销选择 | `swap(&str[l], &str[i])` | 恢复现场，让循环继续 | **结果错乱！** 字符串被永久打乱 |

**如果不做第三步（不撤销）会怎样？** 这是一个经典的"学会回溯"的试金石。继续上面的例子：

```
字符串: "ABC", l=0

i=0: swap(0,0) → "ABC"    → 递归... → swap(0,0) → "ABC" (恢复)
i=1: swap(0,1) → "BAC"    → 递归... → 【忘了 swap(0,1) 恢复！】
     此时字符串仍然是 "BAC"！
i=2: swap(0,2) → "CAB"    ← 注意！原本应该把 str[2]='C' 换到位置 0，
     但因为字符串被上次破坏成了 "BAC"，str[2] 变成了 'C'？不，更糟：
     "BAC" 的 str[0]='B', str[2]='C'，swap(0,2) 后变成 "CAB"
     然后递归内部处理的是 "CAB" 而非正确的 "CBA"
```

**结论**：不撤销会导致下一轮循环在**已经被打乱的字符串**上操作，输出结果完全错乱。

### 递归深度与调用栈

以 `ABC` 为例，递归调用栈的变化过程：

```
时间线 →

main() 调用 permute("ABC", 0, 2)
│
├─ permute(l=0): i=0, swap(0,0)="ABC"
│  ├─ permute(l=1): i=1, swap(1,1)="ABC"
│  │  └─ permute(l=2): l==r → printf("ABC") → 返回
│  │  swap(1,1)="ABC"
│  ├─ permute(l=1): i=2, swap(1,2)="ACB"
│  │  └─ permute(l=2): l==r → printf("ACB") → 返回
│  │  swap(1,2)="ABC"
│  └─ 返回
│  swap(0,0)="ABC"
│
├─ permute(l=0): i=1, swap(0,1)="BAC"
│  ├─ permute(l=1): i=1, swap(1,1)="BAC"
│  │  └─ permute(l=2): l==r → printf("BAC") → 返回
│  │  swap(1,1)="BAC"
│  ├─ permute(l=1): i=2, swap(1,2)="BCA"
│  │  └─ permute(l=2): l==r → printf("BCA") → 返回
│  │  swap(1,2)="BAC"
│  └─ 返回
│  swap(0,1)="ABC"
│
├─ permute(l=0): i=2, swap(0,2)="CBA"
│  ├─ permute(l=1): i=1, swap(1,1)="CBA"
│  │  └─ permute(l=2): l==r → printf("CBA") → 返回
│  │  swap(1,1)="CBA"
│  ├─ permute(l=1): i=2, swap(1,2)="CAB"
│  │  └─ permute(l=2): l==r → printf("CAB") → 返回
│  │  swap(1,2)="CBA"
│  └─ 返回
│  swap(0,2)="ABC"
│
└─ main() 继续...

最大递归深度: 3 层 (l=0 → l=1 → l=2)
对于长度为 n 的字符串，最大递归深度 = n
```

### 复杂度分析

| 指标 | 值 | 说明 |
|------|-----|------|
| **叶子节点数** | n! | 每个叶子代表一个完整排列 |
| **内部节点数** | ≈ n! × (1 + 1/1! + 1/2! + ...) ≈ e × n! | 等比级数求和 |
| **总节点数** | ≈ (e - 1) × n! ≈ 1.718 × n! | 约为叶子数的 1.7 倍 |
| **时间复杂度** | O(n × n!) | n 字符打印 + n! 次递归 |
| **空间复杂度** | O(n) | 递归栈深度 = n |
| **具体数值** | n=8 → 40,320 叶子; n=10 → 3,628,800 叶子 | |

**指数爆炸的直观感受**：

| n | n! | 输出行数 | 大约耗时 |
|---|-----|---------|---------|
| 3 | 6 | 6 | < 1ms |
| 5 | 120 | 120 | < 1ms |
| 8 | 40,320 | 40,320 | ~50ms |
| 10 | 3,628,800 | 360 万行 | ~3s |
| 12 | 479,001,600 | 4.8 亿行 | 太慢了 |

**注意**：这不是算法的问题——全排列本身就有 n! 个输出，任何算法都至少需要 O(n!) 的时间来输出它们。这是问题规模本身决定的，算法已经是最优的（每个排列只用一次 swap + 一次递归调用）。

### 重复字符的处理

当前算法假设输入字符串没有重复字符。如果有重复（如 `AAB`），用同样的算法会输出重复的排列：

```
输入 "AAB"：
  AAB  AAB  ABA  ABA  BAA  BAA

实际上只有 3 种不同的排列: AAB  ABA  BAA

重复原因: 两个 'A' 被当成不同的字符在处理
```

**如何改进？** 在 swap 之前检查：如果 `str[i]` 和当前 `str[l]` 位置的字符相同且不是同一位置，就跳过（因为会产生重复排列）。这是"回溯 + 去重剪枝"的经典模式，也是一个很好的课后延伸练习。

### 重要知识点

- 回溯算法的三要素：**选择**（swap 选一个字符）、**约束**（无，全排列无额外约束）、**目标**（l == r 输出）
- `swap` 作为原地修改的手段：不需要额外空间，直接在字符串上操作
- 撤销操作（backtrack）是整个算法正确性的基石——**每次递归返回后必须恢复现场**
- 递归树的深度 = 字符串长度，每个内部节点有递减的分支数
- 全排列问题是 O(n!) 的，这不是算法的问题，是问题本身的规模

### 课堂讨论

1. **如果字符串有重复字符（如 "AAB"），当前算法会输出重复排列吗？** — 会。两个相同的字符被当作不同的元素。解决方案是在 `swap` 前检查 `i > l && str[i] == str[l]`，跳过重复分支。这是一个经典的"回溯去重"问题。

2. **为什么是 `swap` 而不是新建一个数组？** — `swap` 是原地操作，空间复杂度 O(1)。如果新建数组拷贝剩余字符，每次递归都要 O(n) 的空间和时间，总复杂度会多一个 O(n) 因子。用 `swap` 是最优方案。

3. **如果把 `swap` 的恢复放在递归调用之前会怎样？** — 等于什么都没做。正确的顺序是：swap（改）→ recurse（探索）→ swap（恢复）。

4. **递归和迭代哪个更适合回溯问题？** — 递归更自然，因为回溯天然对应递归树的 DFS 遍历。迭代需要手动维护栈，代码会复杂很多。这正好是递归的强项——"自动管理状态"。

5. **全排列和 DFS 有什么关系？** — 全排列本质上是排列树的 DFS 遍历。排列树是一棵多叉树，每个节点代表一个部分排列。DFS 沿着一条路径走到底（叶子），然后回溯探索其他分支。这是理解 DFS 的绝佳视角。
