## Lesson 36 — 二叉树的前中后序遍历（递归）【重点】

### 课程任务

实现二叉树的三种递归遍历函数：

1. **`preorder(root)`** — 前序遍历：先访问根节点，再递归左子树，最后递归右子树
2. **`inorder(root)`** — 中序遍历：先递归左子树，再访问根节点，最后递归右子树
3. **`postorder(root)`** — 后序遍历：先递归左子树，再递归右子树，最后访问根节点

二叉树已由 `build_tree` 按层序构建好。`make_node` 已提供。三种遍历的递归结构**完全相同**——唯一的区别是**访问根节点的时机**。

### 为什么是重点？

几乎所有的二叉树操作——查找、插入、删除、序列化、反序列化——都以某种遍历顺序访问节点。面试中流畅地写出三种递归遍历是**基本功的最低要求**。如果能进一步解释清楚：①中序遍历 BST 为何得到有序序列；②前序+中序如何唯一还原二叉树，就展现了对树结构的深层理解。

### 二叉树的结构

```c
struct node {
    char ch;                  // 节点存储的字符
    struct node *left;        // 指向左子树的指针（NULL 表示空）
    struct node *right;       // 指向右子树的指针（NULL 表示空）
};
```

每个节点有两个分支指针，形成递归的树形结构。`NULL` 是递归终止条件——遍历到空子树时就返回。

```
        A           ← 根节点 (root)
       / \
      B   C         ← A 的左孩子 B，右孩子 C
     / \   \
    D   E   F       ← B 的左孩子 D、右孩子 E；C 的右孩子 F
   / \
  G   H             ← D 的左孩子 G、右孩子 H

术语:
  - 根节点: 树的最顶层节点 (A)
  - 叶子节点: 无孩子的节点 (E, F, G, H)
  - 父节点: 有孩子的节点 (A 是 B/C 的父)
  - 子树: 以某节点为根的树 (以 B 为根的子树包含 B, D, E, G, H)
```

### 三种遍历的递归模型

三种遍历的函数结构**完全一致**，唯一区别在于 `visit(root)`（访问根）的**位置**：

```
/* 前序: 根 → 左 → 右 */
void preorder(struct node *root) {
    if (root == NULL) return;     // 终止条件: 空树
    printf("%c ", root->ch);      // ① 先访问根
    preorder(root->left);          // ② 递归左子树
    preorder(root->right);         // ③ 递归右子树
}

/* 中序: 左 → 根 → 右 */
void inorder(struct node *root) {
    if (root == NULL) return;
    inorder(root->left);           // ① 先递归左子树
    printf("%c ", root->ch);       // ② 访问根
    inorder(root->right);          // ③ 递归右子树
}

/* 后序: 左 → 右 → 根 */
void postorder(struct node *root) {
    if (root == NULL) return;
    postorder(root->left);         // ① 先递归左子树
    postorder(root->right);        // ② 递归右子树
    printf("%c ", root->ch);       // ③ 最后访问根
}
```

**记忆口诀**：
- **前**序 = 根在最**前面** → 根→左→右
- **中**序 = 根在正**中间** → 左→根→右
- **后**序 = 根在最**后面** → 左→右→根

### 7 节点二叉树的三序遍历

以一棵 7 个节点的完全二叉树为例，演示三种遍历的完整路径：

```
              1              ← 根节点，深度 0
            /   \
          2       3          ← 深度 1
         / \     / \
        4   5   6   7        ← 深度 2（叶子层）
```

#### 前序遍历: 1 → 2 → 4 → 5 → 3 → 6 → 7

```
执行过程（以递归缩进展示调用层次）:

visit(1) → 输出 "1 "                     // 先访问根
├─ preorder(2)
│  ├─ visit(2) → 输出 "2 "              // 访问左子树的根
│  ├─ preorder(4)
│  │  ├─ visit(4) → 输出 "4 "           // 访问叶子
│  │  ├─ preorder(NULL) → return
│  │  └─ preorder(NULL) → return
│  └─ preorder(5)
│     ├─ visit(5) → 输出 "5 "
│     ├─ preorder(NULL) → return
│     └─ preorder(NULL) → return
└─ preorder(3)
   ├─ visit(3) → 输出 "3 "              // 访问右子树的根
   ├─ preorder(6)
   │  ├─ visit(6) → 输出 "6 "
   │  └─ ...
   └─ preorder(7)
      ├─ visit(7) → 输出 "7 "
      └─ ...

输出: 1 2 4 5 3 6 7 ✓
最大递归深度: 3 层 (preorder(1) → preorder(2) → preorder(4))
```

#### 中序遍历: 4 → 2 → 5 → 1 → 6 → 3 → 7

```
执行过程:

inorder(1)
├─ inorder(2)
│  ├─ inorder(4)
│  │  ├─ inorder(NULL) → return
│  │  ├─ visit(4) → 输出 "4 "           // 左子树的左叶子
│  │  └─ inorder(NULL) → return
│  ├─ visit(2) → 输出 "2 "              // 左子树的根（在中间！）
│  └─ inorder(5)
│     ├─ inorder(NULL) → return
│     ├─ visit(5) → 输出 "5 "
│     └─ inorder(NULL) → return
├─ visit(1) → 输出 "1 "                 // 根（在中间！）
└─ inorder(3)
   ├─ inorder(6)
   │  ├─ ...
   │  ├─ visit(6) → 输出 "6 "
   │  └─ ...
   ├─ visit(3) → 输出 "3 "              // 右子树的根
   └─ inorder(7)
      ├─ ...
      ├─ visit(7) → 输出 "7 "
      └─ ...

输出: 4 2 5 1 6 3 7 ✓
```

#### 后序遍历: 4 → 5 → 2 → 6 → 7 → 3 → 1

```
执行过程:

postorder(1)
├─ postorder(2)
│  ├─ postorder(4)
│  │  ├─ postorder(NULL) → return
│  │  ├─ postorder(NULL) → return
│  │  └─ visit(4) → 输出 "4 "           // 左子树的叶子（最后访问！）
│  ├─ postorder(5)
│  │  ├─ ... → visit(5) → 输出 "5 "
│  └─ visit(2) → 输出 "2 "              // 左子树的根（访问完孩子后）
├─ postorder(3)
│  ├─ postorder(6)
│  │  └─ visit(6) → 输出 "6 "
│  ├─ postorder(7)
│  │  └─ visit(7) → 输出 "7 "
│  └─ visit(3) → 输出 "3 "              // 右子树的根
└─ visit(1) → 输出 "1 "                 // 整棵树的根（最后！）

输出: 4 5 2 6 7 3 1 ✓
```

### 不同形状树的三序遍历对比

| 树结构 | 前序 | 中序 | 后序 | 特点 |
|--------|------|------|------|------|
| 单节点 `A` | A | A | A | 三种遍历相同 |
| 左斜 `A→B→C` | A B C | C B A | C B A | 前序=插入序，中后序=逆序 |
| 右斜 `A→B→C` | A B C | A B C | C B A | 前中序相同，后序逆序 |
| 满二叉树 `A(B,C)` | A B C | B A C | B C A | 根在三个不同位置 |
| 一般 `A(B(D,E),C)` | A B D E C | D B E A C | D E B C A | 每种遍历序列都不同 |

**关键观察**:
- 前序遍历的**第一个**元素一定是整棵树的根节点
- 后序遍历的**最后一个**元素一定是整棵树的根节点
- 中序遍历 BST 的结果是**有序的**

### 中序遍历 BST 的特殊性质

对二叉搜索树（BST）做中序遍历，输出是有序的——这是 BST 区别于普通二叉树的**核心性质**：

```
BST 示例:
       5
      / \
     3   7
    / \
   2   4

中序: 2 → 3 → 4 → 5 → 7  ← 从小到大！✓

原因: BST 定义保证 左子树所有值 < 根值 < 右子树所有值
      中序先递归左子树（访问小值），再访问根，最后递归右子树（访问大值）
      天然形成有序序列
```

这也是 Lesson 38 中 BST 操作验证使用中序的原因——插入/删除后中序必须有序。

### 三种遍历的应用场景

| 遍历 | 应用场景 | 原因 |
|------|---------|------|
| 前序 | ① 序列化/反序列化二叉树 ② 打印目录结构 ③ 前缀表达式 | 根在最前，重建时先创建根节点，再递归左右 |
| 中序 | ① BST 有序输出 ② 中缀表达式 ③ 判断 BST 合法性 | BST 的中序天然有序 |
| 后序 | ① 删除整棵树（安全释放） ② 后缀表达式求值 ③ 计算目录大小 | 先处理子节点再处理父节点，安全 |

### 常见错误

| 错误 | 后果 | 正确 |
|------|------|------|
| 忘判 `root==NULL` | 段错误 | 每个遍历函数第一行必须判空 |
| printf 位置放错 | 遍历顺序全错 | 前=最前, 中=两递归间, 后=最后 |
| 三个函数写混 | 输出序列不对 | 用口诀逐行核对 |
| 无限递归（忘推进指针） | 栈溢出 | 确保每次递归都在子树上推进 |

### 重要知识点

- 三种遍历是 DFS（深度优先搜索）在树上的体现——沿一条路径走到底
- `root == NULL` 是递归终止条件——空树是任何树结构的基础情况
- 中序遍历 BST 得到有序序列——这是 BST 区别于普通二叉树的指纹
- 前序+中序（或后序+中序）可以唯一确定二叉树的结构

### 课堂讨论

1. **非递归遍历怎么实现？** — 手动维护一个显式栈（用数组模拟），模拟递归的压栈/弹栈过程。代码更长但避免了递归调用栈溢出的风险。面试中有时要求手写非递归中序遍历。

2. **前序+中序（或后序+中序）能唯一还原二叉树吗？** — 能。前序的第一个元素是根，在中序中找到根的位置，左边是左子树的中序，右边是右子树的中序。递归即可。但前序+后序不能唯一确定。

3. **层序遍历是这三种之一吗？** — 不是。层序是 BFS（广度优先），用队列。前中后序是 DFS（深度优先），用栈（递归隐式）。BFS 和 DFS 是树/图的两种基本搜索策略。

4. **为什么递归遍历二叉树最自然？** — 树的定义是递归的（树 = 根 + 左子树 + 右子树）。递归函数的结构完美匹配树的递归定义。
