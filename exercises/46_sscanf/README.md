## Lesson 46 — sscanf 高级格式化解析

### 课程任务

用 `sscanf` 实现 `parse_addr()` 函数，一行代码解析 `"IP:Port"` 格式字符串，不借助任何正则库：

```c
int parse_addr(const char *input, char *host, int *port);
// 成功返回 1, host 和 port 通过参数返回
// 失败返回 0
```

核心格式符：`%[^:]` —— 匹配"除冒号外的所有字符"，实现轻量级字段提取。

```
验证用例:
  "192.168.1.100:8080\n" → host: 192.168.1.100  port: 8080
  "localhost:3000\n"     → host: localhost      port: 3000
  "invalid\n"            → parse error
```

### 前置知识：sscanf 的格式符体系

```c
int sscanf(const char *str, const char *format, ...);
// 返回值 = 成功匹配并赋值的字段数
```

sscanf 的格式符分为普通格式和高级扫描集：

```
普通格式:                 高级扫描集（字符集）:
  %d  → 整数              %[abc]   → 只匹配 a、b、c 中的字符
  %s  → 字符串(遇空白停)    %[^abc]  → 匹配除 a、b、c 外的所有字符（取反）
  %c  → 单字符             %[0-9]   → 只匹配数字
  %f  → 浮点数             %[a-zA-Z] → 只匹配字母
                           %[^\n]   → 匹配除换行外的所有字符
```

**`%[^:]` 的含义**：`%[` 开启字符集扫描，`^` 表示取反，`:` 是被排除的字符。连起来就是——匹配除 `:` 外的所有字符，遇到 `:` 停止但不消费它。

### 算法详解

```
parse_addr(input, host, port):
  int n = sscanf(input, "%[^:]:%d", host, port);
  return (n == 2) ? 1 : 0;

格式串解析:
  "%[^:]" → 扫描 host: 读所有字符直到遇到 ':'
  ":"     → 匹配并消费字面冒号
  "%d"    → 扫描 port: 读整数
```

#### sscanf 的匹配过程图解

```
输入: "192.168.1.100:8080"
       └─── %[^:] 扫描 ──┘↑└ %d ┘
       host 拿到 "192.168.1.100"
       (读到 ':' 停止，':' 没被消费)
                           ↑
                     ':' 被字面匹配消费
                           port 拿到 8080
返回值: 2 ← host 和 port 都匹配成功 ✓
```

```
输入: "invalid"
       └─── %[^:] 扫描 ──┘
       host 拿到 "invalid"
       然后尝试匹配 ':' → 字符串结束了 → 失败
返回值: 1 ← 只有 host 匹配成功，port 没拿到 → 格式错误!
n != 2 → 返回 0 (解析失败) ✓
```

### 返回值判断模式

```
n == 2 → 两个字段都匹配成功 → 格式正确
n < 2  → 格式不匹配 → 解析失败
```

这就是 C 语言中"在不使用正则库的情况下做格式识别"的标准方法——用 `sscanf` 的返回值判断匹配了几个字段。

### 完整逐步跟踪

#### 例1：正常解析 "192.168.1.100:8080"

```
输入: "192.168.1.100:8080"

sscanf 内部扫描过程:
  格式 "%[^:]:%d"

  ① %[^:] 开始扫描:
     逐个读取字符 '1','9','2','.','1','6','8','.','1','.','1','0','0'
     遇到 ':' → 停止!
     host = "192.168.1.100"
     扫描位置: 冒号处，冒号未消费

  ② 字面 ':' 匹配:
     当前字符是 ':' → 匹配! 消费 ':'
     扫描位置: '8' ('8080' 的开头)

  ③ %d 扫描:
     读取字符 '8','0','8','0' → 数字结束
     port = 8080

  n = 2 (两个字段都匹配) → 返回 1 ✓
```

#### 例2：解析失败 "invalid"

```
输入: "invalid"

sscanf 内部扫描过程:
  ① %[^:] 扫描: 读取 'i','n','v','a','l','i','d'
     遇到 '\0' (字符串结束) → 停止
     host = "invalid"

  ② 字面 ':' 匹配: 当前是 '\0' → 不匹配! 失败

  n = 1 → 返回 0 ✓
```

### 常用高级格式速查

| 格式 | 含义 | 示例输入 | 结果 |
|------|------|---------|------|
| `%[^:]` | 除冒号外所有字符 | `"abc:123"` | `"abc"` |
| `%[^=]` | 除等号外所有字符 | `"key=val"` | `"key"` |
| `%[^,]` | 除逗号外所有字符 | `"a,b,c"` | `"a"` |
| `%[0-9]` | 仅数字 | `"123abc"` | `"123"` |
| `%[^\n]` | 除换行外（读整行） | `"hello\n"` | `"hello"` |

### 字段提取 vs 手动遍历

| 维度 | sscanf | 手动遍历 strchr |
|------|--------|----------------|
| 代码量 | 1 行 | 5-10 行 |
| 可读性 | 格式串即文档 | 需逐步理解 |
| 错误处理 | 返回值检查 | 手动判断 |
| 性能 | 略快（库优化） | 相当 |
| 适用性 | 固定格式 | 更灵活的解析 |

### 常见错误

| 错误 | 后果 | 正确 |
|------|------|---------|
| `%[^:]` 写成 `%s` | `%s` 遇空白/冒号都停 | `%[^:]` 只遇冒号停 |
| 忘记检查 sscanf 返回值 | 解析失败也打印脏数据 | `if (n == 2)` |
| `&port` 写成 `port` | sscanf 不会修改 port | 输出参数必须传地址 |
| 字符集忘了 `^` | `%[:]` 只匹配单个冒号 | 取反必须有 `^` |

### 重要知识点

- `%[^X]` 匹配"除 X 外的所有字符"——`^` = 取反
- `sscanf` 返回成功赋值的字段数——这是判断格式的黄金标准
- sscanf 零依赖，C 标准库自带——简单解析无需正则
- 字符集扫描在遇到不匹配字符时**停止但未消费**该字符

### 课堂讨论

1. **`%s` 和 `%[^:]` 有什么区别？** — `%s` 遇任何空白(空格/tab/换行)停止。`%[^:]` 只遇冒号停止，中间可以有空格。解析 `"hello world:80"` 时 `%s` 只能拿到 `"hello"`，`%[^:]` 能拿到 `"hello world"`。

2. **sscanf 和 strtok 各适合什么场景？** — sscanf 适合已知固定格式的解析（如 `IP:Port`、日期 `YYYY-MM-DD`）。strtok 适合按分隔符切分未知长度的 token 列表。

3. **为什么 `%[^:]` 的 `^` 在方括号里面？** — 方括号定义字符集，`^` 在首位表示取反（不在方括号外面表示行首锚定）。这是扫描集(scan set)语法。

4. **sscanf 能解析 CSV 吗？** — `"%[^,],%[^,],%d"` 可以解析简单的 CSV 行。但复杂的 CSV（引号内有逗号、换行）需要手动状态机。

5. **sscanf 是线程安全的吗？** — 是。与 strtok 不同，sscanf 不修改输入字符串，也没有内部静态变量。每次调用完全独立。

6. **如何防止 `%[^:]` 的缓冲区溢出？** — 加宽度限定：`%63[^:]` 最多读 63 字符（为 `\0` 留 1 字节）。这类似 `snprintf` 的安全边界。

### 实际工程应用场景

```
网络编程:
  sscanf(packet, "%[^:]:%d", ip, &port);
  // 解析 "192.168.1.1:8080" → 连哪个服务器

配置文件解析:
  sscanf(line, "%[^=]=%s", key, value);
  // 解析 "timeout=30" → 读取配置项

日志解析:
  sscanf(log, "%d-%d-%d %d:%d:%d", &y, &m, &d, &H, &M, &S);
  // 解析 "2024-01-15 14:30:00" → 提取时间字段

CSV 解析 (简单场景):
  sscanf(line, "%[^,],%[^,],%d", name, email, &age);
  // 解析 "John,john@mail.com,25" → 提取三个字段
```

### sscanf 格式限定符的高级用法

```
/* 限制读取宽度 */
%10s     → 最多读 10 个字符（防止缓冲区溢出）
%9[^:]   → 最多匹配 9 个非冒号字符

/* 抑制赋值（跳过字段） */
%*d      → 匹配整数但不赋值（吃掉这个字段）
%*[^:]   → 匹配到冒号但不保存

/* 组合使用 */
sscanf("192.168.1.1:8080 extra", "%15[^:]:%d %*s", ip, &port);
→ ip="192.168.1.1", port=8080, "extra" 被跳过
```

### 逐步跟踪：解析非标准格式

```
输入: "key = value" (等号两边有空格)

如果用 sscanf("%[^=]=%s", key, value):
  %[^=] 扫描: "key " (读到 '=', 包含尾部空格!)
  '=' 被消费
  %s 跳过前导空白 → 扫描: "value"
  key = "key " ✗ (有尾部空格!)

修正: sscanf(" key = value ", " %[^= ] = %s", key, value);
  前导空格 ' ' → 跳过空白
  %[^= ] → 匹配到 '=' 或 ' ' 停止 → key = "key" ✓
  然后跳过空白, 字面匹配 '=', 跳过空白
  %s → value = "value" ✓
```

### 常见错误（续）

| 错误 | 后果 | 正确 |
|------|------|---------|
| `%[^:]` 后忘了 `:` | host 吞掉冒号后字符 | `"%[^:]:%d"` |
| `%[^:]` 捕获含尾部空格 | key="key " 而非 "key" | `%[^= ]` 排除空格 |
| 缓冲区溢出（host[64]） | 写穿栈 | `%63[^:]` 限定宽度 |
| 用 `%s` 而非 `%[^:]` | 空格截断 | IP 地址无空格, 但域名可能有 |
