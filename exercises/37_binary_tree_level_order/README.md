## Lesson 37 — 二叉树层序遍历（BFS）

### 课程任务

实现 `levelorder(root)` 函数，用队列辅助进行二叉树的**层序遍历**（Breadth-First Search，广度优先搜索）。按层从上到下、从左到右打印每个节点的字符。

**学以致用**：这道题要求复用 Lesson 35 的环形队列思想——用数组 + front/rear 双指针 + 取模实现辅助队列。唯一变化是队列存储的元素类型从 `int` 变为 `struct node*`。二叉树已由 `build_tree` 按层序构建好。

### 为什么需要层序遍历？

前中后序（DFS）沿一条路径走到底。层序遍历（BFS）逐层扩展，先访问离根近的节点。两者适用于完全不同的场景：

```
        A
       / \
      B   C
     / \
    D   E

层序 (BFS):  A  →  B  →  C  →  D  →  E    (逐层，先近后远)
             第0层  第1层  第1层  第2层  第2层

前序 (DFS):  A  →  B  →  D  →  E  →  C    (一路向下，到底回溯)
```

| 维度 | DFS (前中后序) | BFS (层序) |
|------|---------------|-----------|
| **数据结构** | 栈（递归隐式调用栈） | 队列（显式数组） |
| **策略** | 深度优先：沿一条路径走到底再回溯 | 广度优先：先访问距离近的，逐层扩展 |
| **遍历方式** | 递归函数调用（自动管理状态） | 循环 + 显式维护队列 |
| **首次访问根** | 前序先访问，中序后序较晚 | 立即（第一层第一个） |
| **典型应用** | BST 有序输出、表达式求值、序列化 | 最短路径、层级打印、完全二叉树判定 |
| **空间复杂度** | O(h) 递归栈深度（h=树高） | O(w) 队列大小（w=树最宽层） |

### 算法详解

```
levelorder(root):
  if (root == NULL) return;              // 空树

  struct node *queue[MAX];               // 辅助队列（存节点指针）
  int front = 0, rear = 0;              // 队列的 front/rear 双指针

  queue[rear++] = root;                  // ① 根节点入队

  while (front < rear) {                 // ② 队列非空时循环
    struct node *cur = queue[front++];   // ③ 出队: 取队头
    printf("%c ", cur->ch);             // ④ 访问: 打印节点

    if (cur->left)                       // ⑤ 左孩子入队
      queue[rear++] = cur->left;
    if (cur->right)                      // ⑥ 右孩子入队
      queue[rear++] = cur->right;
  }
```

**核心循环不变的量**: 每次循环开始时，`queue[front..rear-1]` 包含当前层和下一层的所有节点指针，按从左到右的顺序排列。

### 队列状态快照总览

```
轮次 | front | rear | queue[0..4]     | 本轮输出 | 累计输出
-----|-------|------|-----------------|---------|----------
  0  |   0   |   1  | [A]             |    —    | (开始)
  1  |   1   |   3  | [A,B,C]         |    A    | A
  2  |   2   |   5  | [A,B,C,D,E]     |    B    | A B
  3  |   3   |   5  | [A,B,C,D,E]     |    C    | A B C
  4  |   4   |   5  | [A,B,C,D,E]     |    D    | A B C D
  5  |   5   |   5  | [A,B,C,D,E]     |    E    | A B C D E
```

**观察**：
- 第 0 层（A）入队 1 次，出队 1 次
- 第 1 层（B, C）入队 1 次，出队 1 次
- 第 2 层（D, E）入队 1 次，出队 1 次
- 每个节点恰好入队/出队一次 → 时间复杂度 O(n)

### 层级识别技巧

BFS 遍历时如果需要按层分组（如逐行打印），只需在每层开始时记录当前队列大小：

```
while (front < rear) {
    int levelSize = rear - front;   // 当前层有多少个节点
    for (int i = 0; i < levelSize; i++) {
        cur = queue[front++];
        printf("%c ", cur->ch);
        if (cur->left)  queue[rear++] = cur->left;
        if (cur->right) queue[rear++] = cur->right;
    }
    printf("\n");  // 每层结束换行
}

输出变为:
A
B C
D E
```

**原理**：循环开始时 `rear - front` 恰好等于当前层的节点数，因为上一层的所有节点已出队，而下一层的所有节点刚刚入队完毕。

### 队列元素类型的泛型化

Lesson 35 的队列存储 `int`，本题存储 `struct node*`：

```
Lesson 35:  int queue[MAX];
            queue[rear] = val;       // 入队 int

Lesson 37:  struct node *queue[MAX];
            queue[rear++] = root;    // 入队 struct node*
```

指针也是数据，可以放入任何类型的数组。这就是泛型编程思想——队列不关心存储什么类型的数据，只管理"先进先出"的顺序。C++ 的 `std::queue<T>` 和 Java 的 `Queue<T>` 都是这一思想的体现。

### BFS 的关键性质

1. **按层访问**: 距离根相同的节点在同一轮被访问
2. **先近后远**: 先入队的先出队 → 保证先访问离根近的节点
3. **入队顺序决定访问顺序**: 先左后右入队 → 同层从左到右访问
4. **最短路径基础**: BFS 在无权图中的首次相遇就是最短路径——这是 Dijkstra/迷宫求解的理论基础

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 忘判 `root==NULL` | 段错误 | 函数开头判断 |
| 队列未取模 | front/rear 无限增长或越界 | 取模 `(ptr+1)%MAX`（但本题节点数已知，可不取模） |
| 入队顺序左右颠倒 | 访问顺序变为"从右到左" | 先左后右 |
| 队列存储 `char` 而非 `struct node*` | 丢失节点信息，无法访问孩子 | `struct node *queue[MAX]` |

### 重要知识点

- 层序 = BFS，队列是核心辅助——先入先出保证按层访问
- 队列元素类型泛化：`int` → `struct node*`，思想不变
- BFS 保证"先访问离根近的节点"——无权图最短路径的理论基础
- 时间复杂度 O(n)，空间复杂度 O(w)（w=树的最宽层）

### 课堂讨论

1. **BFS 和 DFS 各适合什么场景？** — BFS: 最短路径、层级遍历、社交网络中"一度人脉"；DFS: 路径存在性、回溯问题、拓扑排序。

2. **如何用层序遍历判断一棵树是不是完全二叉树？** — 层序遍历时，遇到第一个 NULL 节点后，后续所有节点的左右孩子也必须是 NULL。如果后面还有非 NULL 孩子，则不是完全二叉树。

3. **队列最大长度是多少？** — 树的最宽层节点数。完全二叉树的最后一层有大约 n/2 个节点。如果树是一条链（退化为链表），队列最大长度为 1——BFS 退化为顺序访问。

4. **为什么 Dijkstra 最短路径用 BFS 思想？** — 无权图中，BFS 的层级就是最短距离。有权图中，用优先队列替代普通队列，每次扩展最近的节点，这就是 Dijkstra 算法的核心。
