## Lesson 41 — 快速排序【重点】

### 课程任务

实现快速排序，包含两个核心函数：

1. **`partition(arr, lo, hi)`** — Lomuto 分区：以 `arr[hi]` 为 pivot，重排数组使 pivot 归位，返回 pivot 索引
2. **`quicksort(arr, lo, hi)`** — 递归快排：分区后对左右子数组递归

```c
int partition(int arr[], int lo, int hi);
void quicksort(int arr[], int lo, int hi);
```

`swap` 已提供。验证用例：
```
输入 "5 3 8 1 4 2\n" → 1 2 3 4 5 8
输入 "9 1 5 3 7\n"   → 1 3 5 7 9
```

快速排序是工程中最常用的原地排序算法，也是 C 标准库 `qsort` 的底层基础。Tony Hoare 1960 年发明它时年仅 26 岁，2011 年获颁 IEEE 冯·诺依曼奖。

### 前置知识：分治与分区

#### 分治（Divide and Conquer）

```
分治三步:
  Divide:  将原问题划分为子问题
  Conquer: 递归求解子问题
  Combine: 合并子问题的解
```

快排的映射：**Divide**=partition 分区，**Conquer**=递归左右，**Combine**=不需要！pivot 归位后左右排好序整体就排好了——这是原地排序的优势。

#### 分区是什么？

选定 pivot（基准值），将数组重排为 `[≤pivot] [pivot] [≥pivot]` 三部分：

```
分区前: [5,  3,  8,  1,  4,  2]     pivot = 2 (最后元素)
分区后: [1] [2] [8, 5, 4, 3]        pivot 在索引1，已归位
         ←≤2→ ↑ ←────≥2────→
```

分区完成后 pivot 处于**最终排序位置**——后续递归只处理左右子数组。

### 算法详解：Lomuto 分区逐步演示

以 `[5, 3, 8, 1, 4, 2]` 为例，pivot = arr[hi] = 2：

```
变量含义:
  pivot = arr[hi] = 2    (基准值)
  i = lo - 1 = -1        (小于等于区的右边界)
  j = lo..hi-1            (扫描指针)

═══════════════════════════════════════════════════
初始: [5, 3, 8, 1, 4, 2]  i=-1, pivot=2
       ↑              ↑
       j              pivot
═══════════════════════════════════════════════════

j=0: arr[0]=5 > 2 → 不交换 (i 保持 -1)
     [5, 3, 8, 1, 4, 2]  i=-1

j=1: arr[1]=3 > 2 → 不交换
     [5, 3, 8, 1, 4, 2]  i=-1

j=2: arr[2]=8 > 2 → 不交换
     [5, 3, 8, 1, 4, 2]  i=-1

j=3: arr[3]=1 ≤ 2 → i++(i=0), swap(arr[0],arr[3])
     [1, 3, 8, 5, 4, 2]  i=0  ← 1 换到最前面!
     ↑        ↑
     i        j

j=4: arr[4]=4 > 2 → 不交换
     [1, 3, 8, 5, 4, 2]  i=0

═══════════════════════════════════════════════════
扫描结束。pivot 归位: swap(arr[i+1], arr[hi]) = swap(arr[1],arr[5])
     [1, 2, 8, 5, 4, 3]  return i+1 = 1
     ←≤2→ ↑ ←──≥2──→
═══════════════════════════════════════════════════

分区结果: pivot=2 在索引1。左边 [1] 全 ≤2，右边 [8,5,4,3] 全 ≥2 ✓
```

### 递归全过程：排序 `[5,3,8,1,4,2]`

```
                     [5,3,8,1,4,2]
                    partition → p=1
                    /              \
               [1]                   [8,5,4,3]
             lo≥hi→返回          partition(pivot=3)→p=2
                                   /           \
                              [2]               [8,5,4]
                            lo≥hi→返回      partition(pivot=4)→p=1
                                              /           \
                                         [3]               [8,5]
                                       lo≥hi→返回    partition(pivot=5)→p=1
                                                        /           \
                                                   [4]               [8]
                                                 lo≥hi→返回    lo≥hi→返回

最终: [1, 2, 3, 4, 5, 8] ✓
```

### Lomuto vs Hoare 分区

| | Lomuto | Hoare |
|---|--------|-------|
| pivot | arr[hi] | arr[lo]（或中间） |
| 交换次数 | 较多（每次都放在 ≤ 区） | 较少（只在两侧都遇到错位时） |
| 返回值 | pivot 最终位置 | 分界点（左 ≤ pivot ≤ 右） |
| 教学 | ✅ 简单直观 | 稍复杂 |
| 本题 | ✅ | — |

### 复杂度分析

| 情况 | 时间 | 空间 | 条件 |
|------|------|------|------|
| 最佳 | O(n log n) | O(log n) | 每次分区近乎二分 |
| 平均 | O(n log n) | O(log n) | 随机输入 |
| 最坏 | O(n²) | O(n) | 已排序 + 选末尾 pivot |

**最坏情况的精确演示**：数组 `[1,2,3,4,5]`，每次选最后一个为 pivot：

```
[1,2,3,4,5] → partition: pivot=5 → p=4 → [1,2,3,4] [5]
[1,2,3,4]   → partition: pivot=4 → p=3 → [1,2,3] [4]
[1,2,3]     → partition: pivot=3 → p=2 → [1,2] [3]
[1,2]       → partition: pivot=2 → p=1 → [1] [2]

每轮 partition 分别扫了 5,4,3,2 个元素 → 总计 14 ≈ n(n+1)/2 → O(n²)
```

**优化方案**：随机 pivot（`swap(arr[random], arr[hi])` 再分区）或三数取中（取 lo/mid/hi 的中位数）。

### 快排 vs 归并 vs 堆排

| 维度 | 快排 | 归并 | 堆排 |
|------|------|------|------|
| 平均时间 | O(n log n) | O(n log n) | O(n log n) |
| 最坏时间 | O(n²) | O(n log n) | O(n log n) |
| 空间 | O(log n) | O(n) | O(1) |
| 稳定 | 否 | 是 | 否 |
| 原地 | 是 | 否 | 是 |
| 缓存友好 | ✅ 极好 | 一般 | ❌ 差 |

### 常见错误

| 错误 | 后果 | 正确 |
|------|------|------|
| i 初始为 lo 而非 lo-1 | 第一个 ≤pivot 的元素不会被交换 | `i = lo - 1` |
| j 扫描到 hi（含 pivot） | pivot 也被交换 | `j < hi`（不含） |
| pivot 归位写成 `swap(i, hi)` | pivot 放错位置 | `swap(i+1, hi)` |
| 递归忘判 `lo >= hi` | 无限递归 | 先判终止条件 |
| 递归左右区间写错 | `p` 被重复排序或遗漏 | 左 `lo..p-1`，右 `p+1..hi` |

### 重要知识点

- Lomuto 分区：i 是 ≤pivot 区的右边界，j 扫描遇到 ≤pivot 的元素时扩容
- pivot 归位后处于最终正确位置——这是快排能原地排序的关键
- 最坏情况 O(n²) 源于每次分区极不平衡——随机化 pivot 可降至 O(n log n)
- 快排是"实际最快"的通用排序——常数小、缓存友好、可原地

### 课堂讨论

1. **如何避免最坏 O(n²)？** — 随机 pivot（swap 随机元素到末尾）、三数取中（lo/mid/hi 的中位数）、切换到堆排（introsort）。

2. **快排稳定吗？为什么？** — 不稳定。partition 过程中相同元素的相对顺序可能被交换打乱。如需稳定，用归并排序。

3. **三路快排是什么？** — 处理大量重复元素时，分成 `<pivot` / `==pivot` / `>pivot` 三区。荷兰国旗问题。

4. **Java 为什么不直接用快排排序引用类型？** — 引用类型需稳定排序，Java 用 TimSort（改进归并排序）。基本类型用 Dual-Pivot Quicksort。

5. **快排和递归深度有什么关系？** — 平衡分区递归深度 O(log n)；退化分区 O(n)，可能栈溢出。生产实现通常先递归小区间，大区间尾递归优化。
