## Lesson 29 — 八皇后问题：回溯 + 剪枝【标杆题】

### 课程任务

在 8×8 的国际象棋棋盘上放置 8 个皇后，使得任意两个皇后不能互相攻击。统计所有可能的摆放方案数量。皇后攻击规则：同行、同列、同对角线（正斜线 / 反斜线）。

实现 `solve(row)` 函数完成逐行放置，输出所有解的个数（应为 92）。`is_safe(row, c)` 冲突检测函数已提供。

### 前置知识：皇后攻击的几何模型

皇后是国际象棋中最强的棋子，她可以沿**横线、竖线、正斜线、反斜线**任意距离移动。理解她的攻击范围是写对冲突检测的前提。

**皇后攻击范围图示**（Q 是皇后，× 是她能攻击到的所有格子）：

```
     0   1   2   3   4   5   6   7    (列)
   ┌───┬───┬───┬───┬───┬───┬───┬───┐
0  │ × │   │   │ × │   │   │   │ × │  反斜线 (↘)
   ├───┼───┼───┼───┼───┼───┼───┼───┤
1  │   │ × │   │ × │   │   │ × │   │
   ├───┼───┼───┼───┼───┼───┼───┼───┤
2  │   │   │ × │ × │   │ × │   │   │
   ├───┼───┼───┼───┼───┼───┼───┼───┤
3  │ × │ × │ × │ Q │ × │ × │ × │ × │  ← 同行攻击
   ├───┼───┼───┼───┼───┼───┼───┼───┤
4  │   │   │ × │ × │   │ × │   │   │
   ├───┼───┼───┼───┼───┼───┼───┼───┤
5  │   │ × │   │ × │   │   │ × │   │
   ├───┼───┼───┼───┼───┼───┼───┼───┤
6  │ × │   │   │ × │   │   │   │ × │  正斜线 (↗)
   └───┴───┴───┴───┴───┴───┴───┴───┘
    ↑
   同列攻击
```

**关键几何公式**：在同一条对角线上的任意两点 (r1,c1) 和 (r2,c2)，满足：

```
|r1 - r2| == |c1 - c2|
```

这就是冲突检测中 `abs(row - i) == abs(c - col[i])` 的几何来源。

### 降维技巧：一维数组表示棋盘

传统思路可能用一个 8×8 的二维数组表示棋盘，但八皇后有一个天然约束：**每行恰好放一个皇后**（否则同行攻击）。既然按行递归，每行只放一个，就只需记录每行的皇后放在哪一列。

**降维后的数据结构**：

```c
int col[8];  // col[row] = 皇后在第 row 行所在的列号 (0~7)
```

**棋盘与数组的映射关系**：

```
实际棋盘 (8×8):                    col[8] 数组:
  0 1 2 3 4 5 6 7                 索引(行)  值(列)
0 . . Q . . . . .                 col[0] = 2
1 . . . . Q . . .                 col[1] = 4
2 . Q . . . . . .                 col[2] = 1
3 . . . . . . . Q                 col[3] = 7
4 Q . . . . . . .                 col[4] = 0
5 . . . Q . . . .                 col[5] = 3
6 . . . . . Q . .                 col[6] = 5
7 . . . . . . Q .                 col[7] = 6

棋盘上 (row, col[row]) 处有皇后
```

**这个技巧为什么成立？**

```
因为: 每行必须放 1 个皇后（按行递归天然保证）
所以: 不需要 8×8=64 个格子的二维数组，8 个元素的数组足够
效果: 冲突检测只需 O(row) 扫描已放置的行，而不是 O(64)
```

### 冲突检测 `is_safe(row, c)` 详解

`is_safe` 函数已提供在模板中，但理解它每行代码的含义至关重要：

```c
int is_safe(int row, int c) {
    for (int i = 0; i < row; i++) {    // 只检查已放置的 0..row-1 行
        if (col[i] == c)               // 同列冲突
            return 0;
        if (abs(row - i) == abs(c - col[i]))  // 同对角线冲突
            return 0;
    }
    return 1;  // 所有检查通过，此位置安全
}
```

**为什么不需要检查 `i > row` 的行？** 因为那些行还没有放置皇后。

**为什么不需要检查行冲突？** 因为按行递归，每行只放一个，天然避免。

**对角线冲突的详细证明**：

```
正斜线 (↗) 示例:                    反斜线 (↘) 示例:
  (1,3) 和 (2,2): 行差 1, 列差 1 ✓   (0,0) 和 (2,2): 行差 2, 列差 2 ✓
  (1,3) 和 (3,1): 行差 2, 列差 2 ✓   (1,4) 和 (3,2): 行差 2, 列差 2 ✓
  (1,3) 和 (4,0): 行差 3, 列差 3 ✓

任意对角线上两点: |r1-r2| == |c1-c2|  恒成立
```

### 算法详解：逐行递归 + 剪枝

#### 核心思路

```
solve(row):
  若 row == 8:           // 所有 8 行都成功放置
    count++               // 找到一个解
    返回

  对每一列 c 从 0 到 7:
    若 is_safe(row, c):   // 检查与已放置的皇后不冲突
      col[row] = c        // 放置
      solve(row + 1)      // 递归下一行
      // 无需显式撤销！因为下一轮循环会覆盖 col[row]
    否则:
      跳过（这就是剪枝！不进入无效分支）
```

**为什么不需要显式撤销 `col[row]`？** 因为当递归返回后，`for` 循环的下一次迭代会用新的 `c` 值覆盖 `col[row]`。而且 `is_safe(row, c)` 只检查 `i < row` 的行，当前行的旧值不影响后续。

#### 剪枝演示：一行中有多少被剪掉？

以第 0 行皇后在列 3（`col[0] = 3`）为例，看第 1 行各列的检测结果：

```
列 0: col[0]=3≠0 ✓, |1-0|=1, |0-3|=3 ≠1 ✓ → 安全，递归第 2 行
列 1: col[0]=3≠1 ✓, |1-0|=1, |1-3|=2 ≠1 ✓ → 安全，递归第 2 行
列 2: col[0]=3≠2 ✓, |1-0|=1, |2-3|=1 =1 ✗ → 对角线冲突！剪掉！
列 3: col[0]=3=3     → 同列冲突！剪掉！
列 4: col[0]=3≠4 ✓, |1-0|=1, |4-3|=1 =1 ✗ → 对角线冲突！剪掉！
列 5: col[0]=3≠5 ✓, |1-0|=1, |5-3|=2 ≠1 ✓ → 安全，递归第 2 行
列 6: col[0]=3≠6 ✓, |1-0|=1, |6-3|=3 ≠1 ✓ → 安全，递归第 2 行
列 7: col[0]=3≠7 ✓, |1-0|=1, |7-3|=4 ≠1 ✓ → 安全，递归第 2 行

8 列中 3 列被剪掉 → 37.5% 的剪枝率仅在第 1 层！
每层剪枝都会产生指数级的节省——被剪掉的分支下面还有 6 层递归。
```

### 剪枝 vs 暴力枚举：量化对比

| 维度 | 暴力枚举 (8^8) | 回溯 + 剪枝 | 提升 |
|------|---------------|------------|------|
| **搜索空间** | 16,777,216 个叶子节点 | ~15,000 个被访问节点 | — |
| **实际访问节点** | 每个可能的放置都要检查 | 只访问合法路径 | 减少约 1000× |
| **剪枝率** | 0% | > 99.9% | 几乎全部剪掉 |
| **耗时** | 不可行（太慢） | 毫秒级 | — |
| **策略** | 生成所有 8^8 种放置再验证 | 边生成边验证，提前剪掉 | — |

### 完整示例：第一个解的逐步跟踪

`solve(0)` 被调用，枚举第 0 行列 0~7。以下展示产生第 1 个解的完整路径：

```
solve(0): col[0]=0 → solve(1)
  ├─ 第1行列0: 同列冲突 ✗
  ├─ 第1行列1: 对角线冲突 ✗  (|1-0|=1, |1-0|=1)
  ├─ 第1行列2: 安全 ✓ → col[1]=2 → solve(2)
  │   ├─ 第2行列0: 对角线冲突 ✗  (|2-0|=2, |0-0|=0≠2...等一下)
  │   │                                          |2-0|=2, |0-1|=1≠2 ✓
  │   │                                          不，看 (0,0) 和 (2,0): 同列! col[0]=0
  │   ├─ 第2行列0: 同列冲突 ✗ (col[0]=0)
  │   ├─ 第2行列1: 对角线冲突 ✗ (|2-1|=1, |1-2|=1)
  │   ├─ 第2行列3: 对角线冲突 ✗ (|2-1|=1, |3-2|=1? 不...)
  │   │              检查与col[0]=(0,0):|2-0|=2,|3-0|=3≠2 ✓
  │   │              检查与col[1]=(1,2):|2-1|=1,|3-2|=1=1 ✗ 冲突!
  │   ├─ 第2行列4: 安全 ✓ → col[2]=4 → solve(3)
  │   │   └─ ... 继续推导...
  │   │      最终: col[3]=1, col[4]=7, col[5]=5, col[6]=3, col[7]=6
  │   │      solve(8) → row==8! count=1 ← 第 1 个解!
```

### 回溯框架对比：全排列 vs 八皇后

| 维度 | 全排列 (L28) | 八皇后 (L29) |
|------|-------------|-------------|
| **搜索空间** | n! 种排列 | 8^8 = 16,777,216 种放置 |
| **约束** | 无重复使用同一元素 | 皇后之间不能互相攻击 |
| **剪枝** | 无（生成所有排列） | 有（冲突即跳过） |
| **冲突检测** | `used[i]` 标记 | `is_safe()` 检查列和对角线 |
| **撤销操作** | 需要显式 `used[i]=0` | 覆盖赋值，无需显式撤销 |
| **递归深度** | n（排列长度） | 8（行数） |
| **叶子节点数** | n! | < n!（剪枝后远少于 8!） |

### 为什么恰好是 92 个解？

```
8 皇后问题共有 92 个不同的解。

如果考虑对称性（旋转 90°/180°/270° 和镜像翻转），
本质上只有 12 个"不同"的解。

所有 92 个解都可以由这 12 个基本解通过旋转变换得到。

例如: col[] = {0, 2, 4, 1, 7, 5, 3, 6} (第 1 个解)
旋转 90° 后变为另一个不同的解。
```

### 常见错误与陷阱

| 错误 | 为什么错 | 正确做法 |
|------|---------|---------|
| 用二维数组 `board[8][8]` | 代码冗余 | 用一维数组 `col[8]` |
| 检查行冲突 | 行冲突由按行递归天然避免 | 只检查列和对角线 |
| 对角线写成 `row-i == c-col[i]`（无绝对值） | 只查了一个方向 | 必须用 `abs(row-i) == abs(c-col[i])` |
| 忘 `#include <stdlib.h>` | `abs()` 在 stdlib.h 中 | 文件开头 include |
| `solve(row+1)` 写在 `if(is_safe)` 外面 | 不论是否冲突都递归 | 只在 is_safe 返回真时递归 |
| 忘初始化 `count = 0` | 多次调用计数会累积 | 在 solve(0) 前 count=0 |

### 重要知识点

- 回溯三要素：**选择**（枚举列）、**约束**（is_safe 检查）、**目标**（row==8）
- 剪枝：在递归树内部节点就排除不可能分支，避免探索整棵子树
- 对角线公式 `abs(row-i) == abs(c-col[i])` 是八皇后的核心
- 一维数组降维：利用问题内在约束减少数据结构维度
- "覆盖即撤销"：递归返回后下次循环覆盖同位置，旧值自然失效

### 课堂讨论

1. **N 皇后问题**：棋盘扩大到 N×N，算法直接适配。N=8 有 92 解，N=15 有 2,279,184 解。大 N 下如何高效求解是并行计算的经典课题。

2. **如何打印具体棋盘？** 拿到 `col[]` 后两层循环：外层 row、内层 col，`col[row]==c` 处打 `Q` 否则 `.`。是很好的延伸练习。

3. **为什么 is_safe 只检查 `i < row`？** row 之后还没放置，不存在冲突。

4. **如何只找第一个解就停止？** 找到一个解后 `exit(0)` 或用全局标志位，立即从所有递归层返回。

5. **12 个基本解**：92 个解 → 考虑旋转/镜像对称 → 只剩 12 个本质不同解。这是组合数学的经典问题。
