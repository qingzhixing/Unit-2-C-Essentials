## Lesson 34 — 栈实现括号匹配【重点】

### 课程任务

用栈判断字符串中的括号 `() [] {}` 是否全部正确配对。实现 `check_brackets(s)` 函数，匹配返回 1，不匹配返回 0。栈的基本操作（`is_empty`、`push`、`pop`）和配对判断（`is_match`）已由模板提供。

```
int check_brackets(const char *s);

验证用例:
  "()\n"       → yes     (基本配对)
  "()[]{}\n"   → yes     (多种括号)
  "([)]\n"     → no      (交叉不匹配)
  "(\n"        → no      (左括号多余)
```

这是数据结构课程中栈应用的经典入门题，也是后续编译原理单元（语法分析、括号匹配）的基石。理解"就近配对 = LIFO"这一核心洞察，你就掌握了括号匹配问题的本质。

### 前置知识：栈的 LIFO 特性与实现

栈（Stack）是一种**后进先出**（LIFO, Last In First Out）的数据结构。像一摞盘子：最后放上去的最先取下来。

```
操作序列: push(A) → push(B) → push(C) → pop() → pop()

栈状态变化:
  [ ]      [A]      [A]      [A]      [A]      [ ]
                      [B]      [B]      [B]
                               [C]

pop() 总是返回栈顶元素——即最后 push 的那个
```

本题用数组实现栈：

```c
#define MAX 1024
char stack[MAX];
int top = -1;           // -1 表示空栈

int  is_empty(int top) { return top == -1; }
void push(char *stack, int *top, char c) { stack[++(*top)] = c; }
char pop(char *stack, int *top) { return stack[(*top)--]; }
```

**`top` 指针的三种状态**：

| top 值 | 含义 | 栈内容示例 |
|--------|------|-----------|
| -1 | 空栈 | `[]` |
| 0 | 1 个元素 | `['(']` |
| k | k+1 个元素 | `['(','[','{']` |

### 为什么括号匹配天然适合栈？

括号匹配有一个**"就近配对"**的特性——每个右括号必须与**最近未配对的左括号**配对。这正是 LIFO 的语义！

```
字符串: "{[()]}"

'{' → push → stack = ['{']                         (遇到左括号，入栈)
'[' → push → stack = ['{','[']                     (遇到左括号，入栈)
'(' → push → stack = ['{','[','(']                 (遇到左括号，入栈)
')' → pop → '(' → is_match('(',')')=1 → 配对! ✓    (右括号，与栈顶配对)
      stack = ['{','[']                            (栈顶被弹出)
']' → pop → '[' → is_match('[',']')=1 → 配对! ✓
      stack = ['{']
'}' → pop → '{' → is_match('{','}')=1 → 配对! ✓
      stack = []

遍历结束，栈空 → 全部匹配！→ yes ✓
```

**核心洞察**：最后遇到的左括号就是最先需要被右括号匹配的那个。如果你尝试用队列（FIFO）或数组遍历，要么算法复杂度飙升，要么根本无法正确处理嵌套匹配。

### 算法流程

```
check_brackets(s):
  初始化栈 stack[MAX], top = -1

  遍历 s 的每个字符 ch:
    ┌─ 若 ch 是 '(' 或 '[' 或 '{':
    │    push(ch)                           // 左括号入栈，等待配对
    │
    └─ 若 ch 是 ')' 或 ']' 或 '}':
         若 is_empty(top):
           return 0                         // 【失败类型1】多了一个右括号
         open = pop()
         若 !is_match(open, ch):
           return 0                         // 【失败类型2】括号类型不配对

  遍历结束:
    若 is_empty(top): return 1              // 成功！全部配对
    否则: return 0                          // 【失败类型3】左括号多余
```

**三类失败模式总览**：

| # | 失败类型 | 检测时机 | 示例 | 含义 |
|---|---------|----------|------|------|
| 1 | 多余右括号 | 遍历中，pop 前栈空 | `"())"` | 右括号比左括号多 |
| 2 | 类型不匹配 | 遍历中，pop 后检查 | `"([)]"` | 右括号类型与最近左括号不对 |
| 3 | 多余左括号 | 遍历结束后栈非空 | `"(()"` | 左括号比右括号多 |

### 完整逐步跟踪

#### 测试 1：正常匹配 `"()[]{}"` → yes

```
         ┌─────┬─────────────────────┬──────────────────┐
         │ 字符 │ 操作                │ 栈状态 (top)      │
         ├─────┼─────────────────────┼──────────────────┤
初始     │     │                     │ [] (top=-1)      │
         │ '(' │ push('(')           │ ['('] (top=0)    │
         │ ')' │ open=pop()='('      │ [] (top=-1)      │
         │     │ is_match('(',')')=1 │                  │
         │ '[' │ push('[')           │ ['['] (top=0)    │
         │ ']' │ open=pop()='['      │ [] (top=-1)      │
         │     │ is_match('[',']')=1 │                  │
         │ '{' │ push('{')           │ ['{'] (top=0)    │
         │ '}' │ open=pop()='{'      │ [] (top=-1)      │
         │     │ is_match('{','}')=1 │                  │
         └─────┴─────────────────────┴──────────────────┘
遍历结束，top=-1 → is_empty → yes ✓
```

#### 测试 2：交叉不匹配 `"([)]"` → no

```
         ┌─────┬─────────────────────┬──────────────────┐
         │ '(' │ push('(')           │ ['('] (top=0)    │
         │ '[' │ push('[')           │ ['(','['] (top=1)│
         │ ')' │ open=pop()='['      │                  │
         │     │ is_match('[',')')=0 │ → return 0 no ✗  │
         └─────┴─────────────────────┴──────────────────┘

栈顶是 '['，遇到的是 ')'，不配对！
这说明括号嵌套顺序错误——应该先关 ']' 再关 ')'。
```

#### 测试 3：多左括号 `"(()"` → no

```
         ┌─────┬─────────────────────┬──────────────────┐
         │ '(' │ push('(')           │ ['(']            │
         │ '(' │ push('(')           │ ['(','(']        │
         │ ')' │ pop → '('           │ ['(']            │
         │     │ is_match('(',')')=1 │                  │
         └─────┴─────────────────────┴──────────────────┘
遍历结束，top=0 → 非空 → no ✗

栈中剩一个 '(' 没有配对的右括号。
```

#### 测试 4：多右括号 `"())"` → no

```
         ┌─────┬─────────────────────┬──────────────────┐
         │ '(' │ push('(')           │ ['(']            │
         │ ')' │ pop → '('           │ []               │
         │     │ is_match=1          │                  │
         │ ')' │ is_empty(top)=1     │ → return 0 no ✗  │
         └─────┴─────────────────────┴──────────────────┘

第二个 ')' 时栈已经空了——没有左括号可以配对。
```

### 测试 5：空字符串 `""` → yes

```
空字符串 → 循环不执行 → 遍历结束栈空(top=-1) → yes ✓
```

### 测试 6：只有左括号 `"((("` → no

```
'('→push,'('→push,'('→push → 遍历结束栈非空(top=2) → no ✗
```

### 括号配对映射

| 左括号 | 右括号 | ASCII |
|--------|--------|-------|
| `(` | `)` | 0x28, 0x29 |
| `[` | `]` | 0x5B, 0x5D |
| `{` | `}` | 0x7B, 0x7D |

### 为什么不用 ASCII 差值判断？

部分面试官会问：能不能用 `abs(open-close) <= 2` 简化配对判断？可以，但不推荐——`is_match` 显式判断更清晰、更易扩展（如增加 `<>` 或 `«»` 等非 ASCII 邻近的括号对）。

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 忘判栈空就 pop | 数组 `stack[-1]` 越界 | `if(is_empty(top)) return 0;` |
| 遍历后忘判栈空 | 多余左括号被漏掉 | `return is_empty(top);` |
| is_match 只检查 `()` | `[] {}` 永远返回 no | 必须检查三种括号对 |
| 非括号字符影响 | 算法可能被干扰 | 忽略非括号字符（或明确说明只处理括号） |

### 重要知识点

- 括号匹配 = LIFO = 栈 —— 抽象数据类型匹配现实问题的典范
- 三类失败覆盖所有不匹配情况，缺一不可
- 时间复杂度 O(n)，每个字符入栈/出栈最多一次
- 空间复杂度 O(n)，最坏输入全是左括号
- 本算法是编译器语法分析中括号平衡检查的基础

### 课堂讨论

1. **HTML/XML 标签匹配（`<div>...</div>`）能用栈吗？** — 能。但标签是多字符字符串，栈中存储标签名而非单字符，匹配用 `strcmp`。这本质上是同一个算法。

2. **如何扩展支持更多括号类型？** — 只需扩展 `is_match` 函数。核心算法完全不变——这是良好抽象设计的体现。

3. **空字符串算匹配吗？** — 算。无括号就是完美匹配（栈空 → yes）。这与数学上的"空集是任何集合的子集"逻辑一致。

4. **如果输入只有右括号？** — 第一个字符就栈空，立即返回 no。算法天然处理。

5. **栈在这道题中的角色可以替代吗？** — 理论上可以递归（隐式栈），但显式栈更直观。队列或数组遍历无法正确处理嵌套。
