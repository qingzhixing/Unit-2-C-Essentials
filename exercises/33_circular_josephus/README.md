## Lesson 33 — 循环链表与约瑟夫环问题

### 课程任务

实现循环链表的核心操作，并求解经典的约瑟夫环：

1. **`clist_insert(cur, val)`** — 在 cur 之后插入新节点，返回新节点作为 cursor
2. **`clist_delete(item)`** — 删除 item 节点，返回后继（唯一节点时返回 NULL）

N 人围成一圈，每报 K 出列一人，输出淘汰顺序。`make_node` 已提供。

```
验证用例:
  输入 "10 3\n" → 3 6 9 2 7 1 8 5 10 4
  输入 "5 2\n"  → 2 4 1 5 3
```

### 为什么用循环链表？—— 数据结构选择分析

约瑟夫环问题有三类常见解法：

| 维度 | 数组（标记删除） | 数组（元素前移） | 循环链表 |
|------|----------------|-----------------|---------|
| 删除复杂度 | O(1) 标记，后续扫描需跳过 | O(n) 所有后续元素前移 | O(1) 指针修改 |
| 遍历 K 步 | O(K) 逐元素检查存活 | O(K) 直接索引 | O(K) 跟随 next 指针 |
| 单轮总开销 | O(n) 需扫描已被删元素 | O(n) 移位 | O(K) 纯指针跟随 |
| 空间 | O(N) | O(N) | O(N) + 指针字段 |
| 建模直观性 | 需 visited[] 标记数组 | 索引取模绕弯 | **天然映射"围成一圈"** |

**结论**：循环链表是约瑟夫环的"自然"数据结构——"围成一圈"的物理模型就是循环链表。数组方案在 N 大 K 小时退化为 O(n²)，链表每轮删除只需 O(1)。额外指针开销在现代机器上可忽略。

### 循环链表与单链表的区别

循环链表与单链表的**唯一区别**：尾节点的 `next` 指向头节点（而非 NULL），形成闭环。

```
单链表:    [1]→[2]→[3]→NULL       (有终点，next==NULL 标志结束)
循环链表:  [1]→[2]→[3]              (闭合环，无 NULL)
            ↑         ↓
            └─────────┘
```

这一区别引发三个行为变化：

| 方面 | 单链表 | 循环链表 |
|------|--------|---------|
| 终止条件 | `p == NULL` | `p == cur`（回到起点） |
| 遍历方式 | `while (p)` | **必须** `do { ... } while (p != cur)` |
| 空链表 | `head == NULL` | `cursor == NULL` |
| 首节点初始化 | `head` 直接指向 | **自环** `item->next = item` |

### 为何遍历必须 do-while？（重要！）

```c
// ❌ while 版本 — 起点被跳过！
struct node *p = cur;
while (p != cur) {
    process(p);
    p = p->next;
}
// 首次判断 p != cur 为假 → 循环体从未执行 → 起点被跳过！

// ✅ do-while 版本 — 起点也被访问
struct node *p = cur;
do {
    process(p);          // 先执行再判断 → 起点被处理 ✓
    p = p->next;
} while (p != cur);
```

**根因**：在非空循环链表中，任何节点开始遍历，起点和比较目标相同。`while` 先判断再执行会直接跳出；`do-while` 先执行再判断保证起点至少被处理一次。

### 插入操作：自环初始化

#### 第一个节点的自环（最容易被忽视的细节）

```
cursor == NULL → 这是第一个节点！

p = make_node(val);
p->next = p;     ← 关键！指向自己，形成"自环"
return p;
```

```
自环在内存中的样子:

        ┌─────────────┐
        │ struct node  │
        │  data = 1    │
        │  next ───────┼───┐
        └─────────────┘   │
              ↑           │
              └───────────┘
         (next 指向自己)
```

**为什么不能 `p->next = NULL`？** 循环链表的任何地方都不应该有 NULL。如果第一个节点 `next = NULL`，后续遍历会崩溃（找不到起点），删除操作会因找前驱的路上遇到 NULL 而段错误。自环是循环链表的**最小有效状态**。

#### 后续插入

```
当前 cursor 指向某节点，插入 val:

p = make_node(val);
p->next = cur->next;     // 新节点 → cursor 的后继
cur->next = p;            // cursor → 新节点
return p;                 // 新节点 = 新 cursor

图解: 插入前 cur→[A]→... 
      插入后 cur→[A]→[NEW]→...  NEW 成为新 cursor
```

### 构建过程的完整跟踪

```
构建 1→2→3→4→5 的循环链表:

step 1: cursor = clist_insert(NULL, 1)
  首个节点, 自环: [1]↺, cursor → [1]

step 2: cursor = clist_insert(cursor, 2)
  在 [1] 后插入 2: [1]→[2]→[1]→..., cursor → [2]

step 3: cursor = clist_insert(cursor, 3)
  在 [2] 后插入 3: [1]→[2]→[3]→[1]→..., cursor → [3]

step 4: cursor = clist_insert(cursor, 4)
  在 [3] 后插入 4: [1]→[2]→[3]→[4]→[1]→..., cursor → [4]

step 5: cursor = clist_insert(cursor, 5)
  在 [4] 后插入 5: [1]→[2]→[3]→[4]→[5]→[1]→..., cursor → [5]

cursor 当前指向最后插入的 [5]。
要访问第一个节点: cursor = cursor->next → 指向 [1] ✓
```

### 删除操作

#### 唯一节点（特殊处理）

```
判断: item->next == item → 唯一节点！
操作: free(item); return NULL;

检测时机: 删除前先检查，否则找前驱时 while(prev->next!=item)
          在唯一节点上会永远循环（prev->next 就是 item 自己）
```

#### 一般节点的删除

```
找前驱:
  struct node *prev = item;
  while (prev->next != item)    // 不是 while (prev != item)！
      prev = prev->next;

跳过:
  prev->next = item->next;
  next = item->next;
  free(item);
  return next;                 // 返回后继作为新 cursor
```

### 约瑟夫环完整逐轮跟踪：N=5, K=2

```
构建完: cursor→[5], cursor=cursor->next→[1] (移到起点)

══════════════════════════════════════════
轮 1:
══════════════════════════════════════════

淘汰 [2]! 输出 "2 "

淘汰 [4]! 输出 "4 "

══════════════════════════════════════════
轮 3:
══════════════════════════════════════════
从 [5] 走 1 步: cursor → [1]
淘汰 [1]! 输出 "1 "
删除 [1]: 找前驱 [5], 5->next=3, free(1)
cursor = [3]

══════════════════════════════════════════

输出: 2 4 1 5 3 ✓
```

### 数学解法（扩展知识）

约瑟夫环有 O(N) 的递推公式解：

```
f(1, K) = 0
f(N, K) = (f(N-1, K) + K) % N

f(N,K) 返回的是幸存者的 0-based 索引
例: N=5, K=2:
  f(1)=0 → f(2)=(0+2)%2=0 → f(3)=(0+2)%3=2
  → f(4)=(2+2)%4=0 → f(5)=(0+2)%5=2
  幸存者索引=2 → 即第 3 个人(值=3) ✓
```

链表法展示淘汰过程，公式法求最终结果——可以交叉验证。

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 首节点 `next=NULL` | 后续段错误 | 自环: `item->next = item` |
| 遍历用 `while` 而非 `do-while` | 起点被跳过 | `do{...}while(p != cur)` |
| 删唯一节点不判 `item->next==item` | 找前驱死循环 | 先判唯一再 free |
| 前驱查找用 `prev != item` | 永远循环（无 NULL 终点） | `prev->next != item` |
| 构建完忘 `cursor = cursor->next` | 从最后插入点开始淘汰 | 移到第一个节点再开始 |

### 重要知识点

- 自环是循环链表的最小有效状态：`item->next = item`
- 遍历必须 `do-while`：起点也是循环链表的一个节点
- 删除唯一节点是特例：`item->next == item`
- 前驱查找用 `prev->next != item`：循环链表没有 NULL 终点
- cursor 游标模式：始终追踪"当前位置"，方便下一次操作

### 课堂讨论

1. **约瑟夫环有数学公式吗？** — 有递推公式 `f(N)=(f(N-1)+K)%N`，O(N) 时间 O(1) 空间。链表法展示过程，公式法求结果。

2. **K 很大（如 K=10000）怎么优化？** — `K %= 当前剩余人数` 减少无效绕圈。当 K 小于剩余人数时每轮只走 K-1 步。

3. **循环链表 vs 数组模拟约瑟夫环？** — 链表删除 O(1)，数组删除 O(n)。N 大时链表更优。但若 K 极大且 N 小，数组取模更快（O(1) 定位）。

4. **双向循环链表能简化删除吗？** — 能。删除不需要遍历找前驱（有 prev 指针），O(1) 即可。代价是每个节点多一个指针。

5. **循环链表的其他应用？** — 轮询调度（Round-Robin）、多玩家回合制游戏、音乐播放器循环播放列表。
