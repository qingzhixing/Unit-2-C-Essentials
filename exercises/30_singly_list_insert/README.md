## Lesson 30 — 单链表动态插入：头插法与尾插法

### 课程任务

实现单链表的两种基本插入操作：

1. **`insert_head(head, val)`** — 头插法：在链表头部插入新节点，返回新的 head
2. **`insert_tail(head, val)`** — 尾插法：在链表尾部插入新节点，返回 head

理解指针指向的改变、空链表的边界处理，以及两种插入策略的性能差异。

### 前置知识：节点的内存模型

在动手写代码之前，理解链表节点在内存中的样子至关重要。C 中的链表节点是一个结构体：

```c
struct node {
    int data;              // 数据域：存储实际值
    struct node *next;     // 指针域：指向下一个节点
};
```

**单个节点在内存中的布局**：

```
         struct node (malloc 分配在堆上)
        ┌──────────┬──────────┐
        │   data   │   next   │
        │    42    │  ──┼──→  (指向下一个节点，或 NULL)
        └──────────┴──────────┘
         sizeof(struct node) 通常是 16 字节
         (int 4 + padding 4 + pointer 8 = 16)
```

**`make_node(val)` 做了什么？** — 题目已提供此函数，但理解它的内部原理有助于你写出正确的插入函数：

```
make_node(42) 执行过程:

1. malloc(sizeof(struct node))
   ┌──────────┬──────────┐
   │  ??????  │  ??????  │    ← 堆上新分配的内存（内容未初始化）
   └──────────┴──────────┘
   ↑ p

2. p->data = 42;   p->next = NULL;
   ┌──────────┬──────────┐
   │    42    │   NULL   │
   └──────────┴──────────┘
   ↑ p

3. return p;  → 调用者拿到这个指针
```

### 链表的状态：空链表与初始化

在插入任何节点之前，链表从空的状态开始：

```
空链表: head = NULL

   head
   ┌───┐
   │ ●─┼──→ NULL
   └───┘
```

`head` 是指向第一个节点的指针。空链表时 `head == NULL`，这是后续所有操作的基础——**如果你的函数不处理 `head == NULL` 的情况，就会段错误**。

### 头插法 `insert_head(head, val)` 详解

**核心思想**：新节点成为新的头节点，原来的链表接在后面。

**时间复杂度**：O(1) — 不需要遍历。

#### 图解：将 3 头插到链表 1 → 2 前面

```
初始状态:
   head
   ┌───┐     ┌────┬────┐    ┌────┬────┐
   │ ●─┼──→  │ 1  │ ●──┼─→ │ 2  │NULL│
   └───┘     └────┴────┘    └────┴────┘

步骤 1: 创建新节点 p (值为 3)
          ┌────┬────┐
          │ 3  │NULL│
          └────┴────┘
          ↑ p

步骤 2: p->next = head;  (新节点指向原链表头)
          ┌────┬────┐    ┌────┬────┐    ┌────┬────┐
          │ 3  │ ●──┼─→ │ 1  │ ●──┼─→ │ 2  │NULL│
          └────┴────┘    └────┴────┘    └────┴────┘
          ↑ p             ↑ head (还没改)

步骤 3: return p;  (新节点成为新的 head)
        调用者用 head = insert_head(head, 3) 更新 head:

   head
   ┌───┐     ┌────┬────┐    ┌────┬────┐    ┌────┬────┐
   │ ●─┼──→  │ 3  │ ●──┼─→ │ 1  │ ●──┼─→ │ 2  │NULL│
   └───┘     └────┴────┘    └────┴────┘    └────┴────┘
```

**完整示例**：依次头插 1, 2, 3

```
初始: head = NULL  (空链表)

head = insert_head(head, 1):   1 → NULL

head = insert_head(head, 2):   2 → 1 → NULL

head = insert_head(head, 3):   3 → 2 → 1 → NULL       ← 最终链表

输出: 3 2 1
```

**关键观察**：头插法的结果是**逆序**的！因为每次新元素都插入在最前面。

### 尾插法 `insert_tail(head, val)` 详解

**核心思想**：遍历到链表末尾，将新节点挂在最后一个节点后面。

**时间复杂度**：O(n) — 需要遍历整个链表找到尾部。

#### 图解：将 3 尾插到链表 1 → 2 末尾

```
初始状态:
   head
   ┌───┐     ┌────┬────┐    ┌────┬────┐
   │ ●─┼──→  │ 1  │ ●──┼─→ │ 2  │NULL│
   └───┘     └────┴────┘    └────┴────┘
                              ↑ 这是尾节点: next == NULL

步骤 1: 创建新节点 p (值为 3)
          ┌────┬────┐
          │ 3  │NULL│
          └────┴────┘
          ↑ p

步骤 2: 遍历找到尾节点 (next == NULL 的节点)
          cur 从头走到尾:
   head
   ┌───┐     ┌────┬────┐    ┌────┬────┐
   │ ●─┼──→  │ 1  │ ●──┼─→ │ 2  │NULL│
   └───┘     └────┴────┘    └────┴────┘
    ↑                         ↑ cur (cur->next == NULL, 找到了)

步骤 3: cur->next = p;  (尾节点指向新节点)
   head
   ┌───┐     ┌────┬────┐    ┌────┬────┐    ┌────┬────┐
   │ ●─┼──→  │ 1  │ ●──┼─→ │ 2  │ ●──┼─→ │ 3  │NULL│
   └───┘     └────┴────┘    └────┴────┘    └────┴────┘
                              ↑ cur         ↑ p (挂上去了)

步骤 4: return head;  (head 不变)
```

**特殊情况：空链表**

当链表为空（`head == NULL`）时，没有尾节点可以挂。此时新节点就是全部链表：

```
初始: head = NULL

head = insert_tail(head, 42):

   没有节点可遍历 → 新节点就是整个链表

   head
   ┌───┐     ┌────┬────┐
   │ ●─┼──→  │ 42 │NULL│
   └───┘     └────┴────┘

   return 新节点 (因为 NULL 时新节点就是 head)
```

**完整示例**：依次尾插 1, 2, 3

```
初始: head = NULL

head = insert_tail(head, 1):   1 → NULL

head = insert_tail(head, 2):   1 → 2 → NULL
                               ↑      ↑
                              head   新尾节点

head = insert_tail(head, 3):   1 → 2 → 3 → NULL
                               ↑          ↑
                              head      新尾节点

输出: 1 2 3
```

**关键观察**：尾插法的结果是**顺序**的！元素保持插入的先后顺序。

### 头插法 vs 尾插法：全面对比

| 维度 | 头插法 `insert_head` | 尾插法 `insert_tail` |
|------|---------------------|----------------------|
| **时间复杂度** | O(1) | O(n) |
| **空间复杂度** | O(1) | O(1) |
| **结果顺序** | 逆序（后进先出） | 顺序（先进先出） |
| **是否需要遍历** | 不需要 | 需要遍历到尾部 |
| **空链表处理** | 新节点 → head = 新节点 | 新节点 → head = 新节点 |
| **返回什么** | 新节点（新 head） | 原 head（或新节点，当链表空时） |
| **类似数据结构** | 栈（Stack） | 队列（Queue） |
| **常见用途** | 反转数据、临时缓冲 | 保持原始顺序、日志追加 |

### 两种方法的内存操作对比

理解指针赋值是链表的精髓。下面用地址视角展示两种方法的关键区别：

```
假设节点地址:
  节点1: 0x100, 节点2: 0x200, 新节点: 0x300

头插 insert_head(head, 3) — 只改 1 次指针:
  p->next = head;        // p(0x300)->next = 0x100
  return p;              // 返回 0x300

尾插 insert_tail(head, 3) — 遍历后改 1 次指针:
  cur->next = p;         // cur(0x200)->next = 0x300
  return head;           // 返回 0x100 (head 不变)
```

### 常见错误与陷阱

| 错误代码 | 为什么错 | 正确做法 |
|---------|---------|---------|
| `item->next = head; return item;` 但调用者写 `head->next = ...` | 调用方式错误 | 调用者必须写 `head = insert_head(head, val)` 用返回值更新 head |
| 尾插法忘判空链表 | 段错误！`cur->next = p` 时 cur 可能为 NULL | 先 `if(head==NULL) return p;` |
| 尾插法返回新节点（非空时） | 链表头丢了，之前的节点泄露 | 非空时 return head |
| `malloc` 后忘记 `free` | 内存泄漏。虽然本题 main 已调用 `list_free` | 养成写 `free` 的习惯 |

### 重要知识点

- `malloc` / `free` 动态内存管理：堆上分配，用完释放
- 结构体指针 `->` 操作符：访问指针指向的结构体成员
- `p->next` 是指针赋值，不是拷贝节点 —— 链表的本质就是指针跳转
- 空链表边界：任何操作前先考虑 `head == NULL` 的情况
- 返回值设计：头插返回新 head（因为 head 变了）；尾插返回原 head（除非空链表）

### 课堂讨论

1. **如果尾插也想做到 O(1)？** — 需要额外维护一个 tail 指针，始终指向最后一个节点。这样插入尾部时直接用 `tail->next = p; tail = p;`。这是工程中"链表 + 尾指针"的常见模式。

2. **头插法和栈有什么关系？** — 头插法的"后进先出"(LIFO)行为完全等价于栈。事实上，用链表的头插/头删就可以实现一个栈，而不需要数组。

3. **哨兵节点（sentinel）能简化什么？** — 如果在链表前面放一个"假的头节点"（不存数据），则所有插入/删除操作都不需要特判空链表，代码可以统一。C++ 的 `std::list` 内部就用了哨兵节点。

4. **如果 `malloc` 失败了怎么办？** — 返回 NULL。生产代码中需要检查 `malloc` 的返回值，并在失败时做清理（释放已分配的内存）。本练习不要求，但面试中这是一个加分点。

5. **为什么 `insert_head` 返回 `struct node *` 而不直接修改 head？** — 这是 C 语言的值传递特性决定的。如果你写成 `void insert_head(struct node *head, int val)`，函数内部的 `head = p` 只修改了局部副本，不影响调用者的 head。通过返回值传递新 head 是最简洁的 C 语言方式。
