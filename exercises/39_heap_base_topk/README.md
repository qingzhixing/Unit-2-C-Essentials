## Lesson 39 — 堆与 Top-K 问题

### 课程任务

实现堆数据结构并解决经典的 Top-K 问题——从海量数据中找出最大（或最小）的 K 个数：

1. **`heapify(arr, n, i)`** — 向下调整（小顶堆：让节点 i "下沉"到正确位置）
2. **`build_heap(arr, n)`** — 建堆：从最后一个非叶节点开始，自底向上 heapify
3. **`top_k(arr, n, k)`** — Top-K：输出 arr 中最大的 k 个数

`swap` 辅助函数已提供。

### 前置知识：堆的数组表示

堆是一棵**完全二叉树**——除了最后一层，其他层都是满的。完全二叉树可以用数组紧凑存储，无需指针。

```
数组索引与树节点的对应关系（0-based，本题使用）:

             arr[0]
            /      \
       arr[1]      arr[2]
       /    \       /    \
    arr[3] arr[4] arr[5] arr[6]
     /  \
arr[7] arr[8] ...

对于节点 i:
  父节点:  (i - 1) / 2
  左孩子:  2*i + 1
  右孩子:  2*i + 2
  是否是叶子:  i >= n/2  (后半部分都是叶子)
  最后一个非叶节点:  n/2 - 1
```

```
示例: arr = [3, 1, 5, 8, 2]

索引:  0  1  2  3  4
值:    3  1  5  8  2

树形:
        3(0)
       /    \
      1(1)  5(2)
     /  \
   8(3) 2(4)

验证: arr[0] 的孩子 → arr[1] 和 arr[2] → 值 1 和 5 ✓
      arr[1] 的孩子 → arr[3] 和 arr[4] → 值 8 和 2 ✓
```

### 大顶堆 vs 小顶堆

| | 大顶堆 (Max-Heap) | 小顶堆 (Min-Heap) |
|---|-------------------|-------------------|
| **堆性质** | 父节点 ≥ 所有后代 | 父节点 ≤ 所有后代 |
| **堆顶** | 最大值 | 最小值 |
| **用途** | 堆排序（升序）、优先队列取最大 | 优先队列取最小、Dijkstra |
| **Top-K** | 找最小的 K 个 | **找最大的 K 个** ← 本题 |

```
大顶堆:                小顶堆:
    9                      1
   / \                    / \
  5   7                  3   2
 / \ / \                / \ / \
1  3 2  4              9  5 7  4

堆顶 9 = 最大值         堆顶 1 = 最小值
```

**面试重点**：为什么找最大的 K 个要用**小顶堆**而不是大顶堆？

因为用小顶堆维护 K 个"候选人"——堆顶是 K 个中**最小**的那个。新元素若比堆顶大，说明它可以替换掉当前最小的候选人；若比堆顶小，说明它连门槛都够不着。堆顶就是"动态门槛"。

### 算法详解：heapify（向下调整/下沉）

`heapify(arr, n, i)` 是堆的核心操作。前提：节点 `i` 的左右子树**已经**是合法的堆，但 `i` 本身可能违反堆性质。`heapify` 把 `i` "下沉"到正确位置。

#### 小顶堆 heapify 逐步跟踪

```
初始: arr = [5, 1, 8, 3, 2], n=5, heapify(arr, 5, 0)

树:
        5(0)           ← i=0, 左右子树[1,8] 和 [3,2] 都已是合法小顶堆
       /    \
     1(1)   8(2)
    /  \
  3(3) 2(4)

步骤 1: 找 i=0 与其孩子中的最小值
  left=1 (arr[1]=1), right=2 (arr[2]=8)
  最小值: min(5, 1, 8) = 1 → smallest = 1

步骤 2: smallest ≠ i → 交换 arr[0] ↔ arr[1]
        1(0)
       /    \
     5(1)   8(2)     ← 5 下沉到位置 1
    /  \
  3(3) 2(4)

步骤 3: 递归 heapify(arr, 5, 1) — 5 继续下沉
  left=3 (arr[3]=3), right=4 (arr[4]=2)
  最小值: min(5, 3, 2) = 2 → smallest = 4
  交换 arr[1] ↔ arr[4]:
        1(0)
       /    \
     2(1)   8(2)
    /  \
  3(3) 5(4)

步骤 4: 递归 heapify(arr, 5, 4) — 5 已是叶子，停止

结果: [1, 2, 8, 3, 5] — 合法小顶堆 ✓
      堆顶 1 = 最小值
```

### 算法详解：build_heap（建堆）

关键洞察：从 `n/2 - 1`（最后一个非叶节点）开始，向前到 0，对每个节点调用 `heapify`。

```
arr = [5, 3, 8, 1, 2], n=5

n/2-1 = 1 → 从 arr[1] 开始

第1步: heapify(arr, 5, 1) — 节点 arr[1]=3
  孩子: arr[3]=1, arr[4]=2 → min=1 < 3 → 交换
  [5, 1, 8, 3, 2]

第2步: heapify(arr, 5, 0) — 节点 arr[0]=5
  孩子: arr[1]=1, arr[2]=8 → min=1 < 5 → 交换
  [1, 5, 8, 3, 2]
  5继续下沉: 孩子arr[3]=3,arr[4]=2 → min=2 < 5 → 交换
  [1, 2, 8, 3, 5] — 小顶堆 ✓
```

**为什么从后往前？** 因为 `heapify` 假设子树已经是合法堆。从后往前保证处理节点 i 时，它的子树已经被处理好了。

**复杂度**：建堆 O(n)，不是 O(n log n)！虽然直观上 n/2 次 heapify × O(log n) = O(n log n)，但大部分节点靠近叶子，下沉高度很小。精确分析得 O(n)。

### Top-K 算法

```
top_k(arr, n, k):
  1. 用前 k 个元素建小顶堆  → 堆顶 = 当前 k 个中的最小值（门槛）
  2. 遍历 arr[k..n-1]:
      若 arr[i] > 堆顶:
        堆顶 = arr[i]     → 替换门槛
        heapify(0)        → 恢复堆性质
      若 arr[i] ≤ 堆顶:
        跳过              → 连门槛都够不着
  3. 堆中 k 个元素就是最大的 k 个
```

#### Top-K 逐步演示

```
arr = [5, 3, 8, 1, 9, 2], k=3（找最大的 3 个）

步骤 1: 用前 3 个建小顶堆
  [5, 3, 8] → build_heap → [3, 5, 8]
  堆顶 = 3 (门槛)

步骤 2: 遍历 arr[3]=1
  1 < 3 → 跳过（不够格）

步骤 3: 遍历 arr[4]=9
  9 > 3 → 替换! arr[0]=9, heapify
  [5, 9, 8] → 堆顶 = 5 (新门槛)

步骤 4: 遍历 arr[5]=2
  2 < 5 → 跳过

最终堆: [5, 9, 8] → 最大的 3 个: 9, 8, 5 ✓
```

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 建堆从 0 到 n-1 正向 | 部分节点未调整 | 从 n/2-1 逆向到 0 |
| Top-K 用大顶堆 | 堆顶是最大值，无法作为门槛 | 找最大 K 个 → 小顶堆；找最小 K 个 → 大顶堆 |
| heapify 忘递归 | 只下沉一层 | 交换后递归 heapify(new_pos) |
| heapify 的 left/right 越界 | 访问 arr[n] | 检查 `left < n` 和 `right < n` |

### 重要知识点

- 堆的数组表示：`2i+1`（左子）、`2i+2`（右子）、`(i-1)/2`（父）
- heapify 前提：子树已是堆，让根下沉
- 建堆从 n/2-1 到 0：自底向上保证前提
- Top-K 核心：小顶堆维护 K 个最大值的"门槛"
- 建堆复杂度 O(n)，整个 Top-K 复杂度 O(n log k)

### 课堂讨论

1. **如果找最小的 K 个，用什么堆？** — 大顶堆。堆顶是 K 个中的最大值（门槛），比它大就丢弃。

2. **10 亿个数找 Top 100，内存放不下怎么办？** — 只维护大小为 100 的小顶堆，数据逐批读入。这就是 Top-K 的威力——空间 O(k)，不需要全量数据驻留内存。

3. **堆排序和快速排序各有什么优劣？** — 堆排序最坏 O(n log n)，快排最坏 O(n²)。但快排常数因子更小，实践中通常更快。堆排序的优势是原地排序且最坏有保证。

4. **为什么建堆 O(n) 而不是 O(n log n)？** — 大部分节点在底层，下沉高度小。数学上，Σ(h/2^h) 收敛于常数，总复杂度 = O(n)。
