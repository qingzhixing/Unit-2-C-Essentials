## Lesson 38 — 二叉搜索树：插入、查找与删除

### 课程任务

实现二叉搜索树（BST）的三个核心操作：

1. **`bst_insert(root, val)`** — 插入：小于往左子树走，大于往右子树走，遇到 `NULL` 创建新节点
2. **`bst_search(root, val)`** — 查找：沿 BST 性质搜索，找到返回节点指针，未找到返回 `NULL`
3. **`bst_delete(root, val)`** — 删除【难点】：三种情况（叶子/单子/双子），`find_min` 已提供

`make_node`、`find_min`（找右子树最小值）、`inorder`（中序打印，用于验证）已提供。

### 前置知识：什么是 BST？

二叉搜索树（Binary Search Tree）的定义：对于任意节点，**左子树所有值 ≤ 根值 ≤ 右子树所有值**。

```
合法的 BST:
       5
      / \
     3   7
    / \   \
   2   4   8

中序遍历: 2 3 4 5 7 8  → 有序！ ← BST 的核心特性

不合法的 BST:
       5
      / \
     3   7
    / \
   2   6       ← 6 在 3 的右子树，但 6 > 5（根）！违反了 BST 性质
```

### BST 的递归特性

BST 的操作天然适合递归——每个子树本身也是 BST：

```
bst_insert(root, val):
  if (root == NULL) return make_node(val);   // 到达空位，创建
  if (val < root->data) root->left = insert(root->left, val);
  else if (val > root->data) root->right = insert(root->right, val);
  else { /* val == data，已存在，忽略或处理重复 */ }
  return root;
```

**为什么返回 `root`？** 递归需要把新节点"挂回"树上。父节点的 `left` 或 `right` 通过返回值来更新。

### 插入与查找（镜像操作）

```
插入:                        查找:
5,3,7 依次插入                查找 4

insert(_, 5):                 search(root=5, val=4):
  5                            4 < 5 → search(left, 4)
 / \                           search(root=3, val=4):
3   7                            4 > 3 → search(right, 4)
                                search(root=NULL) → NULL ✗
                                
insert(_, 4):
  5                              
 / \                            查找 7:
3   7                           7 > 5 → search(right, 7)
 \                              search(root=7, val=7):
  4                               7 == 7 → return root ✓
```

插入和查找的时间复杂度都是 O(h)，h 为树高。平衡 BST 的 h ≈ log n，退化成链表的 h = n。

### 删除 `bst_delete(root, val)` — 三种情况

删除是 BST 操作中最复杂的部分，需要处理三种情况：

#### 情况 1：叶子节点（无孩子）

```
删除 4:
    5
   / \
  3   7         →  free(4), 3的left = NULL
   \
    4

结果:
    5
   / \
  3   7
```

**操作**：直接 `free(root)`，返回 `NULL`。

#### 情况 2：只有一个孩子

```
删除 3:
    5
   / \                         5
  3   7         →  free(3)    / \
   \                         4   7
    4                        (3 被 4 替换)

删除 7:
    5
   / \                         5
  3   7         →  free(7)    / \
     /                       3   8
    8                        (7 被 8 替换)
```

**操作**：保存唯一的孩子 `tmp = root->left ? root->left : root->right`，`free(root)`，返回 `tmp`（孩子提升）。

#### 情况 3：有两个孩子【难点！】

```
删除 5:
       5                      
      / \                     
     3   7         目标: 保持 BST 性质，找谁来替代 5？
    / \ / \
   2  4 6  8

找右子树最小值（6）替代:
       6                      
      / \                     
     3   7        中序: 2 3 4 6 7 8  ← 仍然有序！✓
    / \   \
   2  4   8

为什么是右子树最小值？
- 右子树的所有值 > 被删节点的值
- 右子树的最小值 < 右子树其他值
- 所以它恰好位于被删节点和右子树其他值之间 → 完美替代！
```

**操作**：
1. 找 `succ = find_min(root->right)` — 右子树最小值
2. `root->data = succ->data` — 用 succ 的值替换 root
3. `root->right = bst_delete(root->right, succ->data)` — 递归删除 succ（succ 必然是最左节点，属于情况 1 或 2）

### 逐步跟踪：删除 5

```
初始:
       5
      / \
     3   7
    / \ / \
   2  4 6  8

步骤 1: 被删节点 5 有两个孩子 → 找右子树最小值
  find_min(7): 7→6→NULL(6的left) → 最小值是 6

步骤 2: 替换值
  root->data = 6
       6
      / \
     3   7
    / \ / \
   2  4 6  8    ← 现在有两个 6！需要删除下面那个 6

步骤 3: 递归删除右子树的 6
  bst_delete(7, 6): 6 < 7 → 走左边
  bst_delete(6, 6): 节点 6 是叶子 → free, return NULL
  root->left = NULL
       6
      / \
     3   7
    / \   \
   2  4   8

最终: 中序 = 2 3 4 6 7 8 ✓
```

### 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 双子树时直接 `free(root)` | 左右子树全部丢失 | 右子树最小值替代再递归删除 |
| 忘处理 `val == root->data` 的重复情况 | 重复值被插入多次 | `==` 时忽略或根据需求处理 |
| 单子树判断遗漏一个方向 | 有孩子的节点被当叶子删除 | 用 `if-else if` 覆盖左、右、双三种 |
| 递归删除忘 `return root` | 指针断裂 | 每个分支都 return |
| 插入时遗忘 `val > data` 的判断 | `val == data` 也走右子树，重复插入 | 三个分支：`<` 左、`>` 右、`==` 忽略 |

### 重要知识点

- BST 的核心性质：左 < 根 < 右，中序遍历得有序序列
- 删除三种情况的递进关系：叶子（最简单）→ 单子 → 双子（最复杂）
- 右子树最小值替代法保证了 BST 性质不变
- `find_min` 沿 `left` 走到尽头，复杂度 O(h)

### 课堂讨论

1. **BST 退化成链表时性能是多少？** — 插入/查找/删除都是 O(n)。这是 BST 的最大弱点。自平衡树（AVL、红黑树）通过旋转操作保证 h = O(log n)。

2. **为什么找右子树最小值而不是左子树最大值？** — 都可以！左子树最大值也能完美替代。选择一边即可，效果等价。

3. **中序遍历 BST 有什么特殊性质？** — 输出是有序序列。这是 BST 的"指纹"——任何 BST 的中序一定有序。

4. **插入顺序影响 BST 形状吗？** — 影响巨大。"1,2,3,4,5" 插入得到链表；"3,1,5,2,4" 插入得到平衡树。相同数据、不同顺序 → 不同形状。
